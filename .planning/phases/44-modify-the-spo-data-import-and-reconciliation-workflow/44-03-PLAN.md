<objective>
Implement `import_spo_gifts()` and `import_spo_prayers()` — Steps 2 and 3 of the SPO pipeline.

Purpose: This is the bulk of the data-loading work. Gift import must handle three contact resolution paths (named donor by external_constituent_id, anonymous donor via per-missionary contact, unresolved solicitor skip), extract prayer intentions from the same CSV column, and isolate row errors with savepoints. The prayer-only function provides a rerunnability escape hatch for cases where admin wants to extract prayer intentions without reimporting all gifts.

Output: `import_spo_gifts()` and `import_spo_prayers()` added to `spo_services.py`. Test assertions filled in for `TestImportSpoGifts` stubs.
</objective>

    Add `import_spo_gifts()` to `apps/imports/spo_services.py`:

    ```python
    def import_spo_gifts(file_bytes, filename, uploaded_by, force=False) -> ImportBatch:
        """Step 2: Import SPO gifts CSV, attribute to missionaries, handle anonymous donors."""
    ```

    Implementation:
    1. SHA256 dedup check (same pattern as reconcile_missionaries)
    2. `decode_csv_bytes(file_bytes)` → CSV string
    3. Build `io.StringIO`, `csv.DictReader` after `skip_re_type_label_row`
    4. `_build_header_mapping(headers, SPO_GIFT_HEADER_ALIASES)` → canonical column names
    5. Pre-build missionary lookup: `user_lookup = _build_user_lookup(User.objects.filter(is_active=True))`, `alias_lookup = _build_alias_lookup()`
    6. Per-row (inside `transaction.savepoint()` / `transaction.savepoint_rollback()` on exception):
       a. Get `solicitor_name` → `_match_missionary_name()` → if unresolved or new (no User created here — gifts step requires pre-reconciled missionaries), skip row + add to unmatched_unresolved
       b. Check `is_anonymous` flag and `constituent_id`:
          - If anonymous or blank constituent_id: `contact = _get_or_create_anonymous_contact(missionary)`
          - Else: `contact = Contact.objects.filter(external_constituent_id=constituent_id).first()` → if None, skip + add to contact_not_found
       c. Check for existing Gift by external_id → skip if already exists (idempotency)
       d. Create `Gift(donor_contact=contact, amount_cents=_parse_amount_to_cents(...), gift_date=_parse_date(...), external_id=gift_id)`
       e. Call `_maybe_create_prayer_intention(prayer_description, contact, missionary)`
    7. Build summary JSON: per-missionary breakdown, aggregate counts, unmatched_unresolved list
    8. Update and return ImportBatch with COMPLETED status

    Fill in test assertions:
    - `TestImportSpoGifts.test_gift_attributed_to_missionary`: build CSV bytes with solicitor_name matching a User, call import_spo_gifts, assert Gift.objects.count() == 1 and gift.donor_contact.owner == missionary

    - `TestImportSpoGifts.test_prayer_extracted_from_gift_description`: CSV row with prayer description, assert PrayerIntention.objects.count() == 1
