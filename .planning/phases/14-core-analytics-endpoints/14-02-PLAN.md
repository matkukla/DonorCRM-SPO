**Fix days_stalled in get_stalled_contacts():**

Currently, zero-activity contacts (contacts in journals but with no JournalStageEvent records) show `days_stalled: None`. This violates success criteria #2 (CMON-03) which requires including zero-activity contacts with meaningful data.

1. Add a Subquery to get the earliest journal membership date for the contact:
```python
journal_membership_date = JournalContact.objects.filter(
    contact=OuterRef('pk')
).order_by('created_at').values('created_at')[:1]
```

2. Annotate the base queryset with this membership date:
```python
base_qs = Contact.objects.annotate(
    last_activity_date=Subquery(last_activity),
    journal_membership_date=Subquery(journal_membership_date),
).filter(...)
```

3. In the result building, calculate days_stalled for zero-activity contacts using journal_membership_date:
```python
'days_stalled': (
    (timezone.now() - c.last_activity_date).days
    if c.last_activity_date
    else (timezone.now() - c.journal_membership_date).days
    if c.journal_membership_date
    else None
),
```

This way, zero-activity contacts show how long they've been in a journal without any stage events — the time they've been "neglected."

    ordering = sort_field.asc() if hasattr(sort_field, 'asc') else sort_field
else:

stalled = base_qs.order_by(ordering)[offset:offset + limit]

Note: For days_stalled sorting, the default behavior is `desc` (most stalled first). For `asc`, use `.asc()`. Handle both string field names and expression objects.

4. For `days_stalled` sort specifically: oldest activity date = most stalled. So `asc` on the date field corresponds to `desc` on days_stalled (oldest date = most days stalled). Be careful with the inversion: sorting by `days_stalled desc` means `last_activity_date asc` (nulls first = most stalled).
