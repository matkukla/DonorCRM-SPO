**Create frontend/src/components/mpd/MPDStatsInline.tsx:**

A reusable component that renders 3 Card items showing MPD data. Used on both the admin UserDetail page and the missionary Dashboard.

Props:
```typescript
interface MPDStatsInlineProps {
  currentMpdCap: string | null | undefined
  latestRollForwardBalance: string | null | undefined
  monthsRemainingRf: string | undefined
  isLoading?: boolean
}
```

The component renders 3 Card items following the exact same pattern as existing metrics cards on UserDetail.tsx (Card > CardHeader > CardTitle with text-sm font-medium text-muted-foreground > CardContent > text-2xl font-bold):

1. **MPD Cap** — formatted with formatMPDCurrency(currentMpdCap)
2. **Roll Forward Balance** — formatted with formatMPDCurrency(latestRollForwardBalance)
3. **Months Remaining** — display monthsRemainingRf as-is (it's a string — can be numeric like "24" or "infinite"). Show "--" if empty/undefined.

Loading state: If isLoading, render 3 skeleton cards (div with bg-muted rounded animate-pulse, same height as real cards ~h-24).

Import formatMPDCurrency from "@/api/mpd".

**Important:** This component does NOT fetch its own data. It receives data as props. The parent component (UserDetail or Dashboard) handles data fetching. This allows UserDetail to pass data from the overview endpoint (which has all users' data) and Dashboard to pass data from the /me/ endpoint.
