  <name>Task 1: Add MPD read endpoints (overview, me, uploads)</name>
  <files>apps/imports/views.py, apps/imports/urls.py</files>
  <action>
Add three new view classes to apps/imports/views.py after the existing MPDImportView class:

**1. MPDOverviewView (GET /api/v1/imports/mpd/overview/) — Admin only**
- permission_classes = [permissions.IsAuthenticated, IsAdmin]
- Queries latest MPDSnapshot per active user:
  - Get all active users who have at least one MPDSnapshot
  - For each user, get their most recent snapshot (ordered by upload__created_at descending, take first)
  - Return only 3 fields per user: current_mpd_cap, latest_roll_forward_balance, months_remaining_rf
- Response format: `{ "missionaries": [{ "user_id": str(uuid), "user_name": user.full_name, "current_mpd_cap": str(decimal) or null, "latest_roll_forward_balance": str(decimal) or null, "months_remaining_rf": string }] }`
- Note: DecimalField values MUST be serialized as strings (str(value)) since DRF serializes Decimal to string by default. Frontend will parseFloat.
- Optimize: Use a single query approach. Get distinct user IDs from MPDSnapshot, then for each get latest snapshot. The dataset is small (<50 missionaries) so N+1 is acceptable for simplicity, but prefer prefetch if clean.

**2. MPDMyDataView (GET /api/v1/imports/mpd/me/) — Any authenticated user**
- permission_classes = [permissions.IsAuthenticated]  (NOT IsAdmin — missionaries need this)
- Query: MPDSnapshot.objects.filter(user=request.user).select_related('upload').order_by('-upload__created_at').first()
- If no snapshot exists: return `{ "has_data": false }`
- If snapshot exists: return `{ "has_data": true, "current_mpd_cap": str or null, "latest_roll_forward_balance": str or null, "months_remaining_rf": string }`

**3. MPDUploadHistoryView (GET /api/v1/imports/mpd/uploads/) — Admin only**
- permission_classes = [permissions.IsAuthenticated, IsAdmin]
- Query: MPDUpload.objects.filter(status='completed').order_by('-created_at')[:10]
- Return list of last 10 completed uploads with: id, filename, created_at (isoformat), total_rows, matched_count, unmatched_count

Add URL patterns in apps/imports/urls.py:
- path('mpd/overview/', MPDOverviewView.as_view(), name='mpd-overview'),
- path('mpd/me/', MPDMyDataView.as_view(), name='mpd-my-data'),
- path('mpd/uploads/', MPDUploadHistoryView.as_view(), name='mpd-upload-history'),

Place these AFTER the existing `path('mpd/', ...)` line. Import the new view classes in the imports section at top of urls.py.
