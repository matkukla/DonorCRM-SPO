```python
def parse_pledges_csv(file_content: str, user) -> Tuple[List[dict], List[dict]]:
    """
    Parse pledges CSV and return (valid_records, errors).

    Expected columns: pledge_id, entity_id, fund_id (optional), amount, cadence, status, start_date

    STRICT MODE: If ANY orphan FK references found (entity_id not in Contact or
    fund_id provided but not in Fund), returns empty valid_records and all errors.

    Key differences from parse_transactions_csv:
    - fund_id is OPTIONAL (validate only if provided)
    - cadence and status require enum validation
    - CSV 'cadence' maps to Pledge.frequency field
    - start_date CAN be in future (pledges can start later)

    Args:
        file_content: CSV file content as string
        user: User performing import (for owner-scoped Contact lookup)

    Returns:
        Tuple of (valid_records, errors)
    """
```

Key implementation details:
1. Validate headers (pledge_id, entity_id, amount, cadence, status, start_date required; fund_id optional)
2. First pass: validate row format, validate enums, collect entity_ids and fund_ids
3. Enum validation: cadence.lower() must be in VALID_PLEDGE_FREQUENCIES
4. Enum validation: status.lower() must be in VALID_PLEDGE_STATUSES
5. fund_id: Only add to all_fund_ids set if non-empty
6. Second pass: batch validate FK references exist
   - Contact: `Contact.objects.filter(owner=user, external_id__in=all_entity_ids)`
   - Fund: `Fund.objects.filter(external_id__in=all_fund_ids)` (only if all_fund_ids not empty)
7. If missing FKs found, add errors with row numbers for affected records
8. STRICT MODE: If any FK errors, clear valid_records (return empty list)
9. Limit errors to first 20 (consistent with Phase 8/9/10)
10. Do NOT validate start_date against future (pledges can legitimately start in future)

```python
def import_pledges(records: List[dict], user, import_run) -> Tuple[int, int]:
    """
    Import pledges from parsed records.

    Uses update_or_create because Pledge.external_id has conditional unique
    constraint (globally unique, not owner-scoped like Contact).

    Note: NO Contact stats update needed. Contact.has_active_pledge and
    Contact.monthly_pledge_amount are computed properties that query
    pledges.filter(status='active') directly.

    Args:
        records: List of validated pledge dicts
        user: User performing import (for Contact FK lookup)
        import_run: ImportRun instance to update

    Returns:
        Tuple of (created_count, updated_count)
    """
```

Key implementation details:
1. Pre-fetch Contact objects by external_id (owner-scoped)
2. Pre-fetch Fund objects by external_id (only if any fund_ids provided)
3. Use `update_or_create` for each Pledge (conditional unique constraint on external_id)
4. Map CSV 'cadence' to Pledge.frequency field in defaults
5. Set fund=None if record['fund_id'] is empty
6. Update ImportRun with counts and status
7. Return (created_count, updated_count)
8. NO update_contact_stats_for_import call (pledges use computed properties)

**Reuse existing helpers:**
- `_parse_amount()` for amount validation
- `_parse_date()` for date validation
