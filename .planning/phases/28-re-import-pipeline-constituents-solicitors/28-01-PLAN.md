**Step 2: Create `apps/imports/re_services.py`**

Create the shared RE import service module with these functions:

1. `decode_csv_bytes(file_bytes: bytes) -> str` -- Cascading encoding detection:
   - Try `utf-8-sig` first (handles Excel BOM), then `utf-8`, then `windows-1252` (final fallback, accepts all byte values)
   - Raise `ValueError` if no encoding works (unreachable since windows-1252 accepts all bytes)

2. `check_duplicate_import(file_bytes: bytes, import_type: str) -> ImportBatch | None` -- SHA256 dedup:
   - Hash raw `file_bytes` with `hashlib.sha256().hexdigest()` (hash BEFORE decoding, per research pitfall #4)
   - Query `ImportBatch.objects.get(import_type=import_type, sha256_hash=sha256_hash)`
   - Return existing batch if found, None otherwise

3. `validate_csv_headers(reader_fieldnames: list[str], required_headers: set[str], import_type_label: str) -> None` -- Header validation:
   - Build a case-insensitive mapping of actual headers
   - Check all required headers are present (case-insensitive matching)
   - Raise `ValueError` with descriptive message listing missing headers if any are absent
   - Per CONTEXT.md: "required headers must be present, extras ignored"

4. `normalize_solicitor_name(raw_name: str) -> str` -- Name normalization:
   - Strip whitespace. Return empty string for empty input.
   - If contains comma: treat as "Last, First" -- split on first comma, strip parts, lowercase: `f'{last.lower()}, {first.lower()}'`
   - If no comma: treat as "First Last" -- split on whitespace, last word is last name, rest is first: `f'{last.lower()}, {first.lower()}'`
   - Single-word names stay as-is, lowercased

5. `_build_user_name_lookup() -> dict[str, User]` -- Internal helper:
   - Query all active users: `User.objects.filter(is_active=True)`
   - Build dict mapping `f'{user.last_name.lower()}, {user.first_name.lower()}'` to User object
   - Handle duplicate names: if two users normalize to same key, exclude both (ambiguous match per Phase 24 pattern)

6. `import_re_solicitors(file_bytes: bytes, filename: str, uploaded_by: User) -> ImportBatch` -- Orchestrator:
   - Step 1: SHA256 dedup check via `check_duplicate_import()`. If duplicate, return existing batch immediately.
   - Step 2: Decode with `decode_csv_bytes()`.
   - Step 3: Parse with `csv.DictReader`. Validate headers -- required headers for solicitor: a set of common RE solicitor header names. Use case-insensitive header matching with a mapping dict for aliases:
     - `solicitor_id` / `solid` / `sol_id` / `cnsol_1_01_solicit_id` -> maps to `external_solicitor_id`
     - `solicitor_name` / `name` / `full_name` / `cnsol_1_01_name` -> maps to `raw_name`
     At minimum, require the name field (external_solicitor_id is optional but preferred when present).
   - Step 4: Build user name lookup via `_build_user_name_lookup()`.
   - Step 5: Iterate rows with error collection:
     - Track `row_number` (starting at 2, accounting for header row).
     - Collect errors as list of `{"row": int, "error": str}` dicts.
     - Skip rows with empty name field (log error: "Row N: Missing solicitor name").
     - For each valid row:
       a. Extract external_solicitor_id (if present in CSV).
       b. Normalize name via `normalize_solicitor_name()`.
       c. Dedup within file: if external_solicitor_id matches an already-processed solicitor, skip (increment skipped_count). If no external_id, dedup by normalized_name.
       d. Dedup against database: use `Solicitor.objects.filter(external_solicitor_id=ext_id)` if ext_id present, else `Solicitor.objects.filter(normalized_name=norm_name)`.
       e. If existing solicitor found: skip (increment skipped_count).
       f. If new: create Solicitor with normalized_name and external_solicitor_id.
       g. Auto-link to User: look up normalized_name in user_name_lookup dict. If found, set `solicitor.user = matched_user`.
       h. Track created_count.
       i. Track unlinked solicitors (created but no User match) in a list for the summary.
   - Step 6: Create ImportBatch record with:
     - `import_type=ImportBatchType.RE_SOLICITOR`
     - `status=ImportBatchStatus.COMPLETED` (or FAILED if file-level error)
     - `sha256_hash` from step 1
     - `filename`, `uploaded_by`
     - `total_rows`, `created_count`, `updated_count=0`, `skipped_count`, `error_count`
     - `summary` JSON containing: `{"errors": [...], "unlinked_solicitors": [{"name": "...", "external_id": "..."}]}`
   - Wrap the entire import in `transaction.atomic()` so that if a file-level error occurs mid-processing, nothing is partially committed.
   - For file-level errors (wrong format, missing required headers, unreadable encoding): set status to FAILED, save batch with error in summary, return batch. Do NOT raise exceptions -- the caller needs the batch object to report.

**Important patterns to follow:**
- Use `import logging; logger = logging.getLogger(__name__)` at module level
- Log warnings for ID/name conflicts (per CONTEXT.md: "trust the ID match but log a warning")
- All money fields use cents-based PositiveBigIntegerField (not relevant for solicitors, but note for Phase 29)

**Step 2: Create `apps/imports/management/commands/import_re_solicitors.py`**

Django management command that:
- `help = 'Import RE Solicitor CSV file'`
- Arguments:
  - `file` (positional): Path to CSV file
  - `--owner` (required): Email of the user who runs the import (used as `uploaded_by`)
- `handle()` method:
  1. Look up owner user by email. Raise `CommandError` if not found.
  2. Read file bytes. Raise `CommandError` if file not found.
  3. Check file size <= 10 MB (`MAX_UPLOAD_SIZE = 10 * 1024 * 1024`). Raise `CommandError` if too large.
  4. Call `import_re_solicitors(file_bytes, filename, uploaded_by=owner_user)`.
  5. Check if batch is a duplicate (status == 'duplicate'): print warning and return.
  6. Print summary using `self.stdout.write`:
     - `self.style.SUCCESS(f'Import complete: {batch.filename}')`
     - Created: N, Skipped: N, Errors: N
     - If unlinked solicitors exist: print count and list names
     - If errors exist: print first 10 error messages
  7. Use simple progress: print `f'Processing {filename}...'` before calling service.

**Step 3: Add API endpoint in `apps/imports/views.py`**

Add `RESolicitorImportView` class:
- `permission_classes = [permissions.IsAuthenticated, IsAdmin]`
- `parser_classes = [MultiPartParser]`
- `post()` method:
  1. Validate file is present in `request.FILES['file']`
  2. Check file size <= MAX_UPLOAD_SIZE
  3. Read `file_bytes = file.read()`
  4. Call `import_re_solicitors(file_bytes=file_bytes, filename=file.name, uploaded_by=request.user)`
  5. Return Response with:
     ```python
     {
         'batch_id': str(batch.id),
         'status': batch.status,
         'is_duplicate': batch.status == ImportBatchStatus.DUPLICATE,
         'created_count': batch.created_count,
         'updated_count': batch.updated_count,
         'skipped_count': batch.skipped_count,
         'error_count': batch.error_count,
         'total_rows': batch.total_rows,
         'summary': batch.summary,
     }
     ```
  6. Use HTTP 200 for success and duplicate responses. HTTP 400 for missing file/too large. Follow existing pattern from `ContactImportView`.
