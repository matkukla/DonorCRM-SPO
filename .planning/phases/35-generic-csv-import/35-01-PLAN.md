**2. `import_generic_contacts(file_bytes, filename, uploaded_by, owner, match_by='email')` -> ImportBatch:**

Steps (follow RE pattern):
1. `check_duplicate_import(file_bytes, ImportBatchType.GENERIC_CONTACTS)` -- if duplicate, set status=DUPLICATE, save, return
2. `decode_csv_bytes(file_bytes)` for encoding
3. `csv.DictReader` + `_build_header_mapping(reader.fieldnames, GENERIC_CONTACT_HEADER_ALIASES)`
4. Validate required headers based on match_by:
   - `name`: require first_name AND last_name columns
   - `email`: require email column
   - `external_id`: require external_id column
   Also always require (first_name + last_name) OR organization_name for new contact creation (same as RE import minimum data rule).
5. Iterate rows inside `transaction.atomic()`:
   - Build `row_data` dict from col_map with `_sanitize_field()` on all values
   - Match contact based on `match_by` parameter, all within `owner` scope:
     - `name`: `Contact.objects.filter(owner=owner, first_name__iexact=first_name, last_name__iexact=last_name)`. If multiple matches, skip row with error "Multiple contacts match -- use email or external ID matching"
     - `email`: `Contact.objects.filter(owner=owner, email__iexact=email)`. If no email in row, skip with error.
     - `external_id`: `Contact.objects.filter(owner=owner, external_id=external_id)`. If no external_id in row, skip with error.
   - If matched: call `merge_contact_fields(contact, row_data)` (fill blanks only) and increment `updated_count`
   - If not matched: create new Contact with `owner=owner` and available fields. Require (first_name + last_name) or organization_name for creation; skip row with error if neither present. Increment `created_count`.
   - On any exception per row: collect error `{row: row_number, error: str(e)}`, increment `error_count`
6. Compute sha256_hash, create ImportBatch with import_type=GENERIC_CONTACTS, all counts, summary with errors list, status=COMPLETED (or FAILED if all rows errored)
7. Return ImportBatch

**3. `import_generic_donations(file_bytes, filename, uploaded_by, owner, match_by='email')` -> ImportBatch:**

Steps (follow RE pattern):
1. SHA256 dedup via `check_duplicate_import(file_bytes, ImportBatchType.GENERIC_DONATIONS)`
2. `decode_csv_bytes(file_bytes)`
3. `csv.DictReader` + `_build_header_mapping` with GENERIC_DONATION_HEADER_ALIASES
4. Validate required headers: `amount` and `date` always required. Contact matching columns required based on match_by (same logic as contacts).
5. Iterate rows inside `transaction.atomic()`:
   - Build row_data from col_map with `_sanitize_field()`
   - Parse amount via `_parse_amount_to_cents(row_data['amount'])`. Skip row if zero or negative.
   - Parse date via `_parse_date(row_data['date'])`. Skip row if unparseable.
   - Match contact using same match_by logic as contacts (name/email/external_id within owner scope). If no contact found, skip row with error "No contact found matching '...' -- import contacts first"
   - Create Gift record: `Gift.objects.create(donor_contact=contact, amount_cents=amount_cents, gift_date=date, description=row_data.get('description', ''), owner=owner)`. The Gift post_save signal handles stat recalculation automatically.
   - Optionally match fund if fund_name column present: `Fund.objects.filter(name__iexact=fund_name).first()` and set `fund=matched_fund` on Gift.
   - Increment `created_count`. On error: collect `{row, error}`, increment `error_count`.
6. Create ImportBatch with import_type=GENERIC_DONATIONS, counts, summary, status
7. Return ImportBatch

**Important implementation notes:**
- Use `import hashlib` and compute `sha256_hash = hashlib.sha256(file_bytes).hexdigest()` for ImportBatch creation
- Collect errors as list of `{row: int, error: str}` in same shape as RE imports (for ImportResultBanner compatibility)
- Log import start/completion with `logger.info()`
- Use `from apps.core.utils import get_safe_int_param` is NOT needed here (no query params to parse)
