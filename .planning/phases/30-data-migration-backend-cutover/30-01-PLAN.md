     - Gets Pledge and RecurringGift models via `apps.get_model()`
     - FREQ_MAP: `{'semi_annual': 'semi_annually', 'annual': 'annually'}` (monthly, quarterly pass through)

     - Uses `bulk_create(batch, batch_size=1000, ignore_conflicts=True)`

   - `operations` list calls both RunPython operations with `migrations.RunPython.noop` as reverse

1. Create `apps/gifts/signals.py` modeled on `apps/donations/signals.py`:
   - Import threading, post_save, post_delete, receiver
   - Import `Gift` from `apps.gifts.models`
   - Thread-local signal skip mechanism: `_signal_state = threading.local()`, `disable_gift_signals()`, `enable_gift_signals()`, `_signals_disabled()` -- same pattern as donations
   - `@receiver(post_save, sender=Gift)` handler `update_contact_stats_on_gift_save(sender, instance, created, **kwargs)`:
     - Skip if `_signals_disabled()`
     - Always recalculate: `instance.donor_contact.update_giving_stats()` (NOTE: uses `donor_contact` not `contact`)
     - On `created` only:
       - Set `instance.donor_contact.needs_thank_you = True` and save
       - Create Event with `event_type=EventType.DONATION_RECEIVED` (keep existing event type name per research recommendation -- do NOT rename to avoid orphaning existing events), `title=f'Gift from {instance.donor_contact.full_name}'`, `message=f'${instance.amount_dollars} received'`, `severity=EventSeverity.SUCCESS`, `contact=instance.donor_contact`, `user=instance.donor_contact.owner`
   - `@receiver(post_delete, sender=Gift)` handler `update_contact_stats_on_gift_delete`:
     - Calls `instance.donor_contact.update_giving_stats()`
   - NOTE: No pledge fulfillment logic (RecurringGift has no fulfillment tracking per research)
