  <name>Task 1: Build row validation pipeline for Contact, Donation, and Pledge fields</name>
  <files>

  <action>
    Add the following validation functions to `apps/imports/smartsheet.py`:

    **1. Mapping application function:**
    `def apply_mappings(headers: list[str], rows: list[list[str]], confirmed_mappings: dict) -> list[dict]:`
    - `confirmed_mappings` is `{source_header: target_field}` where target_field uses the model-prefixed names (e.g., 'contact_first_name', 'donation_amount')
    - Convert each row from list-of-values to dict keyed by target_field
    - Only include columns that have a mapping (unmapped columns are ignored)
    - Return list of dicts like `[{'contact_first_name': 'John', 'donation_amount': '100', ...}, ...]`

    **2. Row validation function:**
    `def validate_row(mapped_row: dict, row_num: int, owner) -> list[dict]:`
    - Returns list of error dicts: `{'row_number': int, 'field_name': str, 'error_message': str}`
    - Import and reuse helpers from services.py: `_validate_email`, `_parse_amount`, `_parse_date`
    - Also import `VALID_PAYMENT_METHODS`, `VALID_PLEDGE_FREQUENCIES`, `VALID_PLEDGE_STATUSES` from services.py for enum validation

    **Contact field validation:**
    - `contact_first_name`: required if any contact field is mapped, max 100 chars
    - `contact_last_name`: optional, max 100 chars
    - `contact_email`: optional, but if provided must pass `_validate_email()`
    - `contact_phone`: optional, max 20 chars
    - `contact_address`, `contact_city`: optional, max 255 chars
    - `contact_state`: optional, max 2 chars (US state codes)
    - `contact_zip`: optional, max 20 chars
    - `contact_external_id`: optional, max 100 chars
    - `contact_notes`: optional, no max (TextField)

    **Donation field validation:**
    - `donation_amount`: required if any donation field is mapped, use `_parse_amount()`
    - `donation_date`: required if any donation field is mapped, use `_parse_date()`
    - `donation_fund`: optional, validate fund exists by name (case-insensitive lookup against Fund model)
    - `donation_payment_method`: optional, validate against `VALID_PAYMENT_METHODS`
    - `donation_external_id`: optional, max 100 chars

    **Pledge field validation:**
    - `pledge_amount`: required if any pledge field is mapped, use `_parse_amount()`
    - `pledge_frequency`: required if any pledge field is mapped, validate against `VALID_PLEDGE_FREQUENCIES` (case-insensitive)
    - `pledge_start_date`: optional, use `_parse_date()`
    - `pledge_end_date`: optional, use `_parse_date()`
    - `pledge_status`: optional, validate against `VALID_PLEDGE_STATUSES` (case-insensitive), default 'active'
    - `pledge_external_id`: optional, max 100 chars

    Helper: `_has_model_fields(mapped_row, prefix)` — returns True if any key starting with prefix has a non-empty value. Used to determine if contact/donation/pledge record should be created for this row.

    **3. Batch validation function:**
    `def validate_mapped_rows(mapped_rows: list[dict], owner) -> tuple[list[dict], list[dict]]:`
    - Iterate ALL rows — no early stopping per locked decision
    - Collect all errors across all rows
    - Return `(valid_rows, all_errors)` where valid_rows have zero errors and all_errors is flat list of error dicts
    - A row is valid only if ALL its fields pass validation

    **4. Error CSV generation:**
    `def generate_error_csv(errors: list[dict]) -> str:`
    - Input: list of `{'row_number': int, 'field_name': str, 'error_message': str}`
    - Output: CSV string with columns: `Row Number, Field Name, Error Message`
    - One row per error (a source row with 3 errors → 3 CSV rows)
    - Use `csv.writer` with `io.StringIO`

  <name>Task 2: Build commit logic with multi-model creation and duplicate handling</name>
  <files>

  <action>
    Add the commit/import logic to `apps/imports/smartsheet.py`:

    **1. Contact creation/dedup:**
    `def _create_or_match_contact(mapped_row: dict, owner, duplicate_strategy: str) -> tuple[Contact | None, list[dict]]:`
    - Extract contact fields from mapped_row (keys starting with 'contact_')
    - If no contact fields have values, return (None, [])
    - **Duplicate detection by email** (per locked decision):
      - If email is non-empty, check `Contact.objects.filter(owner=owner, email=email).first()`
      - If match found, apply duplicate_strategy:
        - `'skip'`: Return existing contact, add no error. Caller tracks as skipped.
        - `'update'`: Update existing contact's non-empty imported fields. Return updated contact.
        - `'flag'`: Do NOT import. Return (None, [error with message "Duplicate contact (email: X) -- flagged for review"])
      - If no email or no match: create new Contact
    - Map target fields to model fields: `contact_first_name` → `first_name`, `contact_address` → `street_address`, `contact_zip` → `postal_code`, etc.
    - Set `owner=owner`
    - Use `_parse_date` for any date fields if applicable
    - Return (contact_instance, errors_list)

    **2. Donation creation/dedup:**
    `def _create_donation(mapped_row: dict, contact: Contact, owner, duplicate_strategy: str) -> tuple[Donation | None, bool, list[dict]]:`
    - Extract donation fields from mapped_row (keys starting with 'donation_')
    - If no donation fields have values, return (None, False, [])
    - **Duplicate detection by external_id only** (per Claude's discretion recommendation):
      - If `donation_external_id` is non-empty, check `Donation.objects.filter(external_id=donation_external_id, contact__owner=owner).first()`
      - If match found and strategy is 'skip': return (existing, True_for_skipped, [])
      - If match found and strategy is 'update': update fields, return (updated, False, [])
      - If match found and strategy is 'flag': return (None, False, [flagged error])
      - If no external_id or no match: always create new
    - Map fields: `donation_amount` → parse with `_parse_amount()`, `donation_date` → parse with `_parse_date()`, `donation_fund` → lookup Fund by name (case-insensitive), `donation_payment_method` → validate
    - Link to contact: `contact=contact`
    - Return (donation_instance, was_skipped, errors_list)

    **3. Pledge creation/dedup:**
    `def _create_pledge(mapped_row: dict, contact: Contact, owner, duplicate_strategy: str) -> tuple[Pledge | None, bool, list[dict]]:`
    - Same pattern as donations but for pledge fields
    - **Duplicate detection by external_id only**
    - Map fields: `pledge_amount` → `_parse_amount()`, `pledge_frequency` → validate enum, `pledge_start_date`/`pledge_end_date` → `_parse_date()`, `pledge_status` → validate enum
    - Link: `contact=contact`
    - Fund lookup if `pledge_fund` is mapped
    - Return (pledge_instance, was_skipped, errors_list)

    **4. Main commit function:**
    `def commit_import(session: ImportSession, confirmed_mappings: dict, duplicate_strategy: str, owner) -> dict:`
    - Load parsed data from session: `session.headers`, `session.rows`
    - Apply mappings: `mapped_rows = apply_mappings(session.headers, session.rows, confirmed_mappings)`
    - Validate all rows: `valid_rows, validation_errors = validate_mapped_rows(mapped_rows, owner)`
    - Use `django.db.transaction.atomic()` for the entire commit block
    - For each valid row:
      1. Create/match contact (if contact fields present)
      2. Create donation linked to contact (if donation fields present, requires contact)
      3. Create pledge linked to contact (if pledge fields present, requires contact)
      4. Track created/updated/skipped/flagged counts
    - If a row has donation/pledge fields but no contact could be resolved (no contact fields AND no email match), add error: "Row N: Donation/Pledge requires a contact (map contact fields or provide email for matching)"
    - Update ImportSession fields: status, imported_count, skipped_count, error_count
    - Store errors in ImportRowError records for error CSV download (reuse existing model)
    - Create ImportRun audit record for the session
    - After import, call `update_contact_stats_for_import()` from services.py if any donations were created (to update denormalized Contact stats)
    - Return result dict: `{imported_count, skipped_count, error_count, total_rows, errors_preview (first 20)}`

    **Important edge cases (from research):**
    - Rows with no email → always create new contacts (duplicate detection doesn't apply)
    - Empty email with `UniqueConstraint(condition=~Q(email=''))` → multiple blank emails are OK
    - Use `select_for_update()` on ImportSession before committing to prevent double-commit race condition (Pitfall 7)
    - Check session status is 'pending_mapping' or 'pending_commit' before allowing commit
  </action>
