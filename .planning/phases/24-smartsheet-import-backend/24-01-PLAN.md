  <name>Task 2: Create smartsheet.py with parsing, format detection, sanitization, and column mapping</name>
  <files>

  <action>
    Create `apps/imports/smartsheet.py` with the following functions:

    **1. Formula sanitization (applied to every cell value during parsing):**
    ```python
    FORMULA_CHARS = {'=', '+', '-', '@', '\t', '\r', '\n'}

    def sanitize_cell_value(value: str) -> str:
        """Strip leading formula injection characters from imported cell values (OWASP)."""
        if not value or not isinstance(value, str):
            return value or ''
        while value and value[0] in FORMULA_CHARS:
            value = value[1:]
        return value.strip()
    ```
    Note: This STRIPS on import (unlike `sanitize_csv_value` in services.py which PREFIXES on export).

    **2. File format detection (magic bytes):**
    ```python
    def detect_file_format(file_bytes: bytes) -> str:
        """Detect xlsx vs csv from magic bytes. Returns 'xlsx' or 'csv'."""
        if len(file_bytes) >= 4 and file_bytes[:4] == b'PK\x03\x04':
            return 'xlsx'
        return 'csv'
    ```

    **3. Excel parsing (openpyxl read_only mode):**
    `def parse_xlsx(file_bytes: bytes) -> tuple[list[str], list[list[str]]]:`
    - Use `openpyxl.load_workbook(io.BytesIO(file_bytes), read_only=True, data_only=True)`
    - First row = headers: `str(cell.value).strip() if cell.value is not None else ''`
    - Handle openpyxl type pitfalls per research:
      - `None` cells → empty string
      - `int/float` cells → `str(cell.value)`
      - `datetime.date/datetime.datetime` cells → `.isoformat()`
      - All other → `str(cell.value).strip()`
    - Apply `sanitize_cell_value()` to every data cell (not headers — headers need original names for mapping)
    - Return `(headers, rows)` where rows is list of list of strings
    - Always close workbook in `finally` block
    - Enforce max 50,000 rows — if exceeded, raise ValueError

    **4. CSV parsing (stdlib csv, utf-8-sig for BOM handling):**
    `def parse_csv(file_bytes: bytes) -> tuple[list[str], list[list[str]]]:`
    - Decode with `utf-8-sig` first, fallback to `latin-1` on UnicodeDecodeError
    - Use `csv.reader(io.StringIO(content))`
    - First row = headers (stripped)
    - Pad short rows with '' to match header count, trim long rows
    - Apply `sanitize_cell_value()` to every data cell (not headers)
    - Return `(headers, rows)`
    - Enforce max 50,000 rows

    **5. Unified parse entry point:**
    `def parse_file(file_bytes: bytes) -> tuple[str, list[str], list[list[str]]]:`
    - Calls `detect_file_format()` then dispatches to `parse_xlsx` or `parse_csv`
    - Returns `(file_format, headers, rows)`

    **6. Column alias registry (COLUMN_ALIASES dict):**
    Define the full alias map per research Pattern 3. Target fields organized by model:
    - **Contact:** first_name, last_name, email, phone, street_address (maps to 'address' context field), city, state, postal_code (maps to 'zip' context field), external_id, notes
    - **Donation:** donation_amount (disambiguated from pledge), donation_date, fund, payment_method, donation_external_id
    - **Pledge:** pledge_amount, frequency, start_date, end_date, pledge_status (disambiguated from other 'status'), pledge_external_id

    Important: Since a single spreadsheet can map to all 3 models, disambiguate overlapping field names by prefixing with model name. Use the following target field naming:
    - `contact_first_name`, `contact_last_name`, `contact_email`, `contact_phone`, `contact_address`, `contact_city`, `contact_state`, `contact_zip`, `contact_external_id`, `contact_notes`
    - `donation_amount`, `donation_date`, `donation_fund`, `donation_payment_method`, `donation_external_id`
    - `pledge_amount`, `pledge_frequency`, `pledge_start_date`, `pledge_end_date`, `pledge_status`, `pledge_external_id`

    Each target has aliases. Examples:
    - `contact_first_name`: ['first name', 'first_name', 'firstname', 'fname', 'given name']
    - `contact_email`: ['email', 'email address', 'e-mail', 'email_address']
    - `donation_amount`: ['amount', 'gift amount', 'donation amount', 'gift_amount', 'total']
    - etc. (use all aliases from research Pattern 3, assigning each to its model-prefixed target)

    For ambiguous aliases like 'amount' or 'external_id' or 'status' that could match multiple models, assign to the most common model (Contact fields win for generic names like 'email'; Donation wins for 'amount' since it's more common; Contact wins for generic 'external_id'). The user can always remap in the UI.

    **7. Exact alias matching:**
    `def _try_alias_match(normalized_header: str, used_targets: set) -> str | None:`
    - Normalize header: lowercase, strip, replace _ with space
    - Check each target's aliases for exact match
    - Skip already-used targets
    - Return target field name or None

    **8. Fuzzy matching with rapidfuzz:**
    `def _try_fuzzy_match(normalized_header: str, used_targets: set) -> tuple[str | None, int]:`
    - Build choices dict from all aliases of unused targets
    - Use `rapidfuzz.process.extractOne` with `fuzz.token_sort_ratio` scorer, `score_cutoff=70`
    - Return `(target_field, score)` or `(None, 0)`

    **9. Suggestion generator:**
    `def _get_suggestions(normalized_header: str, used_targets: set, limit: int = 3) -> list[str]:`
    - Score each available target by best alias match
    - Return top-N target field names sorted by score

    **10. Auto-mapping entry point:**
    `def auto_map_columns(source_headers: list[str]) -> list[dict]:`
    - For each header:
      - Phase 1: Try exact alias match → confidence 'green'
      - Phase 2: Try fuzzy match with score >= 80 → confidence 'yellow'
      - Phase 3: No match → confidence 'red', include top-3 suggestions
    - Return list of `{source, target, confidence, suggestions?}` dicts
    - Track `used_targets` set to prevent duplicate mappings
  </action>
