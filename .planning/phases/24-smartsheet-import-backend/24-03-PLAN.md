    Add two new view classes to `apps/imports/views.py`:

    **1. SmartsheetUploadView (POST — Step 1 of the import flow):**
    ```
    class SmartsheetUploadView(APIView):
        permission_classes = [IsAuthenticated, IsAdmin]
        parser_classes = [MultiPartParser]
    ```

    POST handler:
    - Validate file presence (`request.FILES['file']`)
    - Validate file size: 25 MB max (`25 * 1024 * 1024`). Use a local `SMARTSHEET_MAX_UPLOAD_SIZE` constant (don't change the existing `MAX_UPLOAD_SIZE = 10MB` which is for SPO imports)
    - Do NOT validate file extension — format is detected from magic bytes
    - Read file bytes: `file_bytes = file.read()`
    - Call `parse_file(file_bytes)` from smartsheet.py → returns `(file_format, headers, rows)`
    - Catch `ValueError` from parse_file (e.g., "exceeds 50,000 rows") and return 400
    - Catch `Exception` from openpyxl/csv parsing and return 400 with descriptive error
    - Call `auto_map_columns(headers)` → returns list of mapping dicts with confidence
    - Create `ImportSession` with:
      - `user=request.user`
      - `filename=file.name`
      - `file_format=file_format`
      - `status=ImportSessionStatus.PENDING_MAPPING`
      - `headers=headers`
      - `rows=rows`
      - `auto_mappings=mappings`
      - `total_rows=len(rows)`
      - `expires_at=timezone.now() + timedelta(hours=24)`
    - Return 201 response:
      ```json
      {
        "session_id": "<uuid>",
        "filename": "donors.xlsx",
        "file_format": "xlsx",
        "total_rows": 150,
        "headers": ["First Name", "Last Name", ...],
        "mappings": [
          {"source": "First Name", "target": "contact_first_name", "confidence": "green"},
          {"source": "Gift Amount", "target": "donation_amount", "confidence": "yellow"},
          {"source": "Random Col", "target": null, "confidence": "red", "suggestions": ["contact_notes", ...]}
        ]
      }
      ```

    **2. SmartsheetSessionView (GET — retrieve session details):**
    ```
    class SmartsheetSessionView(APIView):
        permission_classes = [IsAuthenticated, IsAdmin]
    ```

    GET handler:
    - Look up `ImportSession` by `session_id` (UUID from URL)
    - Filter by `user=request.user` (users can only see their own sessions)
    - Return 404 if not found
    - Return session details:
      ```json
      {
        "session_id": "<uuid>",
        "filename": "donors.xlsx",
        "file_format": "xlsx",
        "status": "pending_mapping",
        "total_rows": 150,
        "imported_count": 0,
        "skipped_count": 0,
        "error_count": 0,
        "headers": [...],
        "auto_mappings": [...],
        "created_at": "2026-02-19T..."
      }
      ```
    - Do NOT include `rows` in response (could be very large). Only include headers, mappings, and counts.

    **1. SmartsheetCommitView (POST — Step 2 of the import flow):**
    Add to `apps/imports/views.py`:
    ```
    class SmartsheetCommitView(APIView):
        permission_classes = [IsAuthenticated, IsAdmin]
    ```

    POST handler:
    - Parse request body (JSON, not multipart):
      - `session_id` (required, UUID)
      - `mappings` (required, dict of `{source_header: target_field}`)
      - `duplicate_strategy` (required, one of: 'skip', 'update', 'flag')
    - Validate `duplicate_strategy` is one of the allowed values
    - Look up ImportSession by session_id, filter by `user=request.user`
    - Return 404 if not found
    - Check session status is `PENDING_MAPPING` or `PENDING_COMMIT` — if not, return 409 Conflict with message "This import session has already been committed"
    - Update session: `confirmed_mappings=mappings`, `duplicate_strategy=duplicate_strategy`, `status=PENDING_COMMIT`
    - Call `commit_import(session, mappings, duplicate_strategy, request.user)` from smartsheet.py
    - Return 200 response:
      ```json
      {
        "session_id": "<uuid>",
        "status": "completed",
        "total_rows": 150,
        "imported_count": 142,
        "skipped_count": 5,
        "error_count": 3,
        "errors_preview": [
          {"row_number": 12, "field_name": "contact_email", "error_message": "Invalid email format"}
        ]
      }
      ```
    - If commit_import raises an exception, set session status to FAILED and return 500

    **2. SmartsheetErrorCSVView (GET — download error CSV):**
    ```
    class SmartsheetErrorCSVView(APIView):
        permission_classes = [IsAuthenticated, IsAdmin]
    ```

    GET handler:
    - Look up ImportSession by session_id (UUID from URL)
    - Filter by `user=request.user`
    - Return 404 if session not found
    - Read errors directly from `session.error_rows` (JSONField, list of `{row_number, field_name, error_message}` dicts — populated by commit_import in Plan 02). Do NOT use ImportRowError/ImportRun — those models use a different schema for SPO imports and have no FK to ImportSession.
    - If `session.error_rows` is empty, return 404 with message "No errors found for this import session"
    - Generate error CSV using `generate_error_csv(session.error_rows)` from smartsheet.py
    - Return HttpResponse with `content_type='text/csv'` and `Content-Disposition: attachment; filename="smartsheet_errors_{session_id}.csv"`

    **3. Wire URL patterns in apps/imports/urls.py:**
    Add imports at top:
    ```python

        SmartsheetUploadView,

    ```

    Add URL patterns to `urlpatterns` list:
    ```python

    path('smartsheet/commit/', SmartsheetCommitView.as_view(), name='smartsheet-commit'),
    path('smartsheet/sessions/<uuid:session_id>/', SmartsheetSessionView.as_view(), name='smartsheet-session'),
    path('smartsheet/sessions/<uuid:session_id>/errors/csv/', SmartsheetErrorCSVView.as_view(), name='smartsheet-errors-csv'),
    ```

    **Important notes:**
    - SmartsheetCommitView does NOT use MultiPartParser (it accepts JSON body, not file upload)
    - The error CSV for smartsheet imports uses `generate_error_csv()` from smartsheet.py (Row Number, Field Name, Error Message format) — this is DIFFERENT from the existing `ImportRunErrorsCSVView` which uses original row data + error_message column
    - Add `from datetime import timedelta` and `from django.utils import timezone` to views.py imports if not already present
    - Import `ImportSession`, `ImportSessionStatus` from models
    - Import `parse_file`, `auto_map_columns`, `commit_import`, `generate_error_csv` from smartsheet
