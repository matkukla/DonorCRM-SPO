**Step 2: Create ErrorFallback component**
Create `frontend/src/components/ErrorFallback.tsx` with:
- Import `FallbackProps` type from `react-error-boundary`
- Full-page centered layout using semantic Tailwind tokens (bg-background, text-foreground, text-muted-foreground, bg-primary, text-primary-foreground)
- Heading: "Something went wrong"
- Description: "An unexpected error occurred. Please try refreshing the page."
- Development-only error message display: Show `error.message` in a `bg-muted` pre block only when `import.meta.env.DEV` is true (Vite environment check, NOT `process.env.NODE_ENV`)
- "Refresh Page" button that calls `resetErrorBoundary`
- All colors use semantic tokens so it renders correctly in both light and dark mode

Reference the code example in 21-RESEARCH.md but use `import.meta.env.DEV` instead of `process.env.NODE_ENV === "development"` since this is a Vite app.

**Step 3: Wire ErrorBoundary into App.tsx**
Import `ErrorBoundary` from `react-error-boundary` and `ErrorFallback` from `@/components/ErrorFallback`.

Wrap the app content with `<ErrorBoundary>` INSIDE `ThemeProvider` but OUTSIDE `AuthProvider` and `BrowserRouter`. This ensures:
1. Dark mode works in the error fallback (ThemeProvider is above)
2. Auth/routing errors are caught (AuthProvider/BrowserRouter are below)

```tsx
<ThemeProvider>
  <ErrorBoundary FallbackComponent={ErrorFallback} onReset={() => window.location.reload()}>
    <QueryClientProvider>
      <AuthProvider>
        <BrowserRouter>
          ...
        </BrowserRouter>
      </AuthProvider>
    </QueryClientProvider>
  </ErrorBoundary>
</ThemeProvider>
```

The `onReset` callback does a full page reload to clear any corrupted state.

Do NOT add per-route error boundaries. One root boundary satisfies QAL-10. Per-route boundaries are a future enhancement.
