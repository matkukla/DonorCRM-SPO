**Utility function (services.py):**
Add a `sanitize_csv_value()` function near the existing `FORMULA_PREFIXES` constant (line ~31) in `apps/imports/services.py`:

```python
def sanitize_csv_value(value):
    """Prevent CSV formula injection by prefixing dangerous characters with single quote."""
    if value and isinstance(value, str) and value.startswith(FORMULA_PREFIXES):
        return "'" + value
    return value
```

Note: `FORMULA_PREFIXES = ('=', '+', '-', '@')` should already exist in this file (used for import validation). If not, add it.

**Apply to export_contacts_csv() in services.py:**
Wrap all user-entered string fields with `sanitize_csv_value()`: first_name, last_name, email, phone, street_address, city, state, postal_code, country, notes. Numeric fields (total_given, gift_count) and date fields are safe and do not need sanitization.

**Apply to export_donations_csv() in services.py:**
Wrap string fields: contact name fields, external_id, notes, fund name. Numeric fields (amount) and date fields are safe.

**Apply to StalledContactsCSVView in export_views.py:**
Import `sanitize_csv_value` from `apps.imports.services`. Wrap string fields: full_name, email, owner name, status description, any notes.

**Apply to TeamActivityCSVView in export_views.py:**
Import `sanitize_csv_value` from `apps.imports.services`. Wrap string fields: user_name, event_type, title, contact_name, message.

The single-quote prefix is the OWASP-recommended approach. Spreadsheet applications interpret the leading `'` as a text-mode indicator and strip it from display. This is NOT visible to the end user in the spreadsheet.
