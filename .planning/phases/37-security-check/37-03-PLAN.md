<task type="auto">
  <name>Task 1: Run automated security scans and fix findings</name>
  <files>
    requirements/dev.txt
  </files>
  <action>
    **Install scanning tools (dev dependencies only):**

    1. Add `bandit` and `pip-audit` to `requirements/dev.txt` (NOT base.txt -- these are dev/CI tools, not production dependencies).
    2. Run `pip install bandit pip-audit` to install them.

    **Run bandit (Python static security analysis):**

    3. Run: `bandit -r apps/ --exclude "*/tests/*,*/migrations/*" -f json -o /tmp/bandit-report.json --severity-level medium`
    4. Review findings. For each finding:
       - **True positive:** Fix the code issue.
       - **False positive in production code:** Add inline `# nosec` comment with explanation.
       - **Test-only issue:** Already excluded via `--exclude`.
    5. Re-run bandit to confirm fixes.

    **Run pip-audit (Python dependency CVE scan):**

    6. Run: `pip-audit --requirement requirements/base.txt --requirement requirements/prod.txt`
    7. If CVEs found:
       - Check if the CVE is relevant to DonorCRM's usage of the library.
       - If fixable: update the version constraint in the appropriate requirements file.
       - If not fixable (e.g., no patch available): document as accepted risk with justification.
    8. Also run: `pip-audit` (without --requirement, to scan installed packages) for completeness.

    **Run npm audit (frontend dependency CVE scan):**

    9. Run: `cd frontend && npm audit --audit-level moderate`
    10. If vulnerabilities found:
        - Run `npm audit fix` for automatic fixes (non-breaking).
        - For remaining issues: check if they affect production (dev dependencies are lower priority).
        - Document any unfixable issues as accepted risk.
    11. Also run: `cd frontend && npm audit --omit=dev` for production-only scan.

    **Git history secrets scan:**

    12. Run these searches against git history:
        ```bash
        git log --all -S "password" --oneline | head -20
        git log --all -S "secret_key" --oneline | head -20
        git log --all -S "api_key" --oneline | head -20
        git log --all -S "PRIVATE_KEY" --oneline | head -20
        git log --all --diff-filter=A -- "*.env" --oneline
        git log --all --diff-filter=A -- "*.pem" --oneline
        git log --all --diff-filter=A -- "*.key" --oneline
        ```
    13. Review any findings. If actual secrets were committed:
        - They should be rotated (note this for the user -- cannot be done by Claude).
        - Document in SECURITY-REPORT.md.
    14. Verify `.gitignore` covers: `.env`, `*.pem`, `*.key`, `secrets.json`, `credentials.json`, `.mcp.json`

    **Fix dependency versions if needed:**

    15. If pip-audit or npm audit flag fixable CVEs, update version constraints in `requirements/base.txt`, `requirements/prod.txt`, or `frontend/package.json` as appropriate.
    16. Run `pip install -r requirements/base.txt` and/or `cd frontend && npm install` to apply fixes.
  </action>
