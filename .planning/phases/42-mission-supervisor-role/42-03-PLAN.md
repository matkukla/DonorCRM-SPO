  **Update `openEditDialog`** to initialize the supervised user IDs:
  The user list from `getUsers()` includes a `supervisor` field. To populate the multi-select, we need to find which users have `supervisor === editingUser.id`. Add this logic:
  ```typescript
  const openEditDialog = (user: User) => {
    setEditingUser(user)
    setEditFirstName(user.first_name)
    setEditLastName(user.last_name)
    setEditRole(user.role)
    setEditError("")
    // Find users supervised by this user
    const supervised = users?.filter(u => u.supervisor === user.id).map(u => u.id) || []
    setEditSupervisedUserIds(supervised)
  }
  ```

  **Update `handleUpdate`** to include supervised_user_ids when role is mission_supervisor:
  ```typescript
  const handleUpdate = async () => {
    if (!editingUser) return
    setEditError("")
    try {
      const data: UserUpdate = {
        first_name: editFirstName,
        last_name: editLastName,
        role: editRole,
      }
      // Include supervised_user_ids when role is mission_supervisor
      if (editRole === 'mission_supervisor') {
        data.supervised_user_ids = editSupervisedUserIds
      }
      await updateMutation.mutateAsync({ id: editingUser.id, data })
      setEditingUser(null)
    } catch {
      setEditError("Failed to update user.")
    }
  }
  ```

  **Add missionary assignment multi-select to edit dialog:**
  After the Role dropdown in the edit dialog, conditionally render a missionary assignment section when `editRole === 'mission_supervisor'`:

  ```tsx
  {editRole === 'mission_supervisor' && (
    <div className="space-y-2">
      <Label>Assigned Missionaries</Label>
      <p className="text-xs text-muted-foreground">
        Select missionaries this supervisor can view
      </p>
      <div className="border rounded-md max-h-48 overflow-y-auto p-2 space-y-1">
        {users?.filter(u => u.id !== editingUser?.id && u.is_active).map(u => (
          <label
            key={u.id}
            className="flex items-center gap-2 px-2 py-1.5 rounded hover:bg-muted cursor-pointer text-sm"
          >
            <input
              type="checkbox"
              checked={editSupervisedUserIds.includes(u.id)}
              onChange={(e) => {
                if (e.target.checked) {
                  setEditSupervisedUserIds(prev => [...prev, u.id])
                } else {
                  setEditSupervisedUserIds(prev => prev.filter(id => id !== u.id))
                }
              }}
              className="rounded border-input"
            />
            <span>{u.full_name}</span>
            <span className="text-muted-foreground ml-auto">{u.email}</span>
          </label>
        ))}
      </div>
      {editSupervisedUserIds.length > 0 && (
        <p className="text-xs text-muted-foreground">
          {editSupervisedUserIds.length} missionary{editSupervisedUserIds.length !== 1 ? 'ies' : ''} assigned
        </p>
      )}
    </div>
  )}
  ```

  This implements the user decision: "Multi-select dropdown with search for the missionary picker when editing a supervisor" and "Only users with mission_supervisor role see the assignment picker on their edit form."

  Note: The user decision says "multi-select dropdown with search". A scrollable checkbox list with all users is a simple starting point. If the user list is small (typical missionary org has <50 users), this is sufficient. For larger orgs, a Command/Combobox pattern could be used, but that's over-engineering for now. Use the checkbox list approach.
