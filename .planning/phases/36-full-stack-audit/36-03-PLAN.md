Fix 3 backend performance issues identified in research:

**A. get_support_progress() in apps/dashboard/services.py (~line 120):**
Currently loads all active RecurringGifts into Python to sum `monthly_equivalent`:
```python
active_recurring = RecurringGift.objects.filter(...)
total = sum(rg.monthly_equivalent for rg in active_recurring)
```
Replace with SQL aggregation using Django's `Case`/`When` for the frequency multiplier:
```python
from django.db.models import Case, When, Value, F, DecimalField, Sum

FREQUENCY_MULTIPLIERS = {
    'weekly': Decimal('4.333'),
    'biweekly': Decimal('2.167'),
    'monthly': Decimal('1'),
    'bimonthly': Decimal('0.5'),
    'quarterly': Decimal('0.333'),
    'semiannually': Decimal('0.167'),
    'annually': Decimal('0.083'),
}

monthly_total = RecurringGift.objects.filter(
    donor_contact__owner=user,
    status='Active'
).annotate(
    multiplier=Case(
        *[When(frequency=freq, then=Value(mult)) for freq, mult in FREQUENCY_MULTIPLIERS.items()],
        default=Value(Decimal('1')),
        output_field=DecimalField(max_digits=10, decimal_places=3)
    ),
    monthly_cents=F('amount_cents') * F('multiplier')
).aggregate(total=Sum('monthly_cents'))['total'] or 0
```
Check the exact frequency values used in the RecurringGift model (look at the `FREQUENCY_CHOICES` or similar) and match them exactly.

**B. get_giving_summary() in apps/dashboard/services.py (~line 200):**
Same pattern -- loads active RecurringGifts into Python. Apply the same SQL aggregation approach.

**C. MPDOverviewView in apps/insights/views.py:**
Research says it queries MPDSnapshot per user in a loop (~lines 697-712). Refactor to use a single query with `.values('user').annotate(...)` or `Distinct` on user to avoid the N+1 pattern. Read the actual code to determine the correct approach.

**D. Missing indexes check:**

- `PrayerIntention.status` (used in filtered queries)
If missing, add `db_index=True` on those fields and create a migration. If they already exist via Meta.indexes or db_index=True, document that they're present.
