**Step 2: Add shared gift import helpers to re_services.py**

Add these functions BELOW the existing constituent import code in `apps/imports/re_services.py`:

1. `_parse_amount_to_cents(amount_str: str) -> int` -- Parse dollar amount string ("$1,234.56", "1234.56", etc.) to cents integer. Remove $, commas, whitespace. Use `Decimal` for precision: `int(Decimal(cleaned) * 100)`. Return 0 for empty/unparseable. Import `Decimal` and `InvalidOperation` from decimal module.

2. `_parse_date(date_str: str) -> date | None` -- Try parsing with multiple formats: `%Y-%m-%d`, `%m/%d/%Y`, `%m/%d/%y`. Return None if all fail. Import `datetime` module.

3. `_build_fund_lookup() -> dict[str, Fund]` -- Build dict mapping lowercased external_id and lowercased name to Fund objects. Import Fund from apps.imports.models. Used for flexible fund matching during gift import.

4. `_build_solicitor_lookup() -> dict[str, Solicitor]` -- Build dict mapping normalized_name to Solicitor objects. Query all Solicitor.objects.all().

5. `_group_rows_by_id(reader: csv.DictReader, col_map: dict, id_field: str) -> tuple[dict[str, list[dict]], list[dict], int]` -- Group CSV rows by gift ID into `defaultdict(list)`. For each row: build `row_data` dict from `col_map` (same pattern as constituent import), extract the gift ID from `id_field` canonical name. Track `_row_number` on each row_data dict (string). Return `(groups_dict, errors_list, total_rows)`. Import `defaultdict` from collections.

6. `PRAYER_STOPLIST` set -- Small conservative stoplist: `{'n/a', 'na', 'none', 'no', 'yes', '-', '--', '---', '...', 'test', 'x', 'xx', 'xxx', 'general', 'same', 'same as above', 'see above', 'ditto', 'tbd', 'unknown'}`

7. `_maybe_create_prayer_intention(gift: Gift, prayer_text: str, contact: Contact, seen_prayers: dict) -> PrayerIntention | None` -- Per CONTEXT.md decisions:
   - Strip text. Return None if empty.
   - Lowercase check against PRAYER_STOPLIST. Return None if match.
   - Sanity check: skip if no alphanumeric chars (`not any(c.isalnum() for c in text)`).
   - Normalize for dedup: `text.lower().strip()`. Dedup key: `(contact.id, normalized)`.
   - If dedup_key in seen_prayers dict: add gift to existing M2M (`existing.gifts.add(gift)`), return existing.
   - Check database: `PrayerIntention.objects.filter(contact=contact, description__iexact=text).first()`. If found, add gift to M2M, cache in seen_prayers, return.
   - Create new: title = `text[:80].rsplit(' ', 1)[0] if len(text) > 80 else text`. If text is >80 chars and rsplit returns the original text (no space found), use `text[:80]` instead. Set status=PrayerIntentionStatus.ACTIVE. After create, call `prayer.gifts.add(gift)`. Cache in seen_prayers.
   - Import PrayerIntention and PrayerIntentionStatus from apps.prayers.models.

**Step 3: Add GIFT_HEADER_ALIASES and import_re_gifts orchestrator**

Add `GIFT_HEADER_ALIASES: dict[str, str]` mapping lowercase aliases to canonical field names. Use the alias set from 29-RESEARCH.md (covers gift_id, constituent_id, amount, gift_date, fund, description, solicitor_name, credit_amount, prayer_description). The prayer column alias MUST include `'gift specific attributes prayer requests description': 'prayer_description'` per locked CONTEXT.md decision.

Add `GIFT_REQUIRED_CANONICAL = {'gift_id', 'constituent_id', 'amount'}` -- minimum headers needed.

Add `import_re_gifts(file_bytes: bytes, filename: str, uploaded_by: User, owner: User) -> ImportBatch`:

Follow the exact same orchestrator pattern as `import_re_solicitors()` and `import_re_constituents()`:

1. Compute SHA256 hash. Check for duplicate via `check_duplicate_import(file_bytes, ImportBatchType.RE_GIFT)`. If duplicate, set status=DUPLICATE, return.
2. Decode via `decode_csv_bytes(file_bytes)`. On failure, create FAILED ImportBatch.
3. Parse CSV with `csv.DictReader`. Build `col_map` via `_build_header_mapping(fieldnames, GIFT_HEADER_ALIASES)`.
4. Validate required canonical fields are mapped. If missing, create FAILED ImportBatch with descriptive error listing expected aliases.
5. Group rows by gift_id using `_group_rows_by_id(reader, col_map, 'gift_id')`.
6. Build lookups: `_build_solicitor_lookup()`, `_build_fund_lookup()`.
7. Process each gift group inside `transaction.atomic()`:
   - For each `(gift_id, rows)` in groups:
     - Use a **savepoint** (`transaction.savepoint()`) per gift group so one failure doesn't roll back the whole import.
     - First row provides gift-level data.
     - Look up Contact by `external_constituent_id` from the `constituent_id` field. If not found, skip entire group per CONTEXT.md locked decision. Log error with all row numbers in group.
     - Check if `Gift.objects.filter(external_gift_id=gift_id).exists()` -- if duplicate against database, skip group, increment skipped_count. (Prevents IntegrityError per Pitfall 3 in research.)
     - Parse amount via `_parse_amount_to_cents()` from the first row's `amount` field. If 0, log error and skip group.
     - Parse date via `_parse_date()` from the first row's `gift_date` field. If None, log error and skip group.
     - Match fund: get `fund` field from first row, look up in fund_lookup (try external_id first, then name, both lowercased). Set to None if not found (not fatal, log warning).
     - Create `Gift` with `external_gift_id=gift_id`, `donor_contact=contact`, `amount_cents=amount_cents`, `gift_date=parsed_date`, `fund=matched_fund`, `description=first_row.get('description', '')`.
     - For each row in the group: extract `solicitor_name`. If empty, skip credit. Normalize via `normalize_solicitor_name()`. Look up in solicitor_lookup. If not found, log error (credit skipped per discretion recommendation). If found, parse `credit_amount` via `_parse_amount_to_cents()`. If credit_amount is 0, default to the gift's amount_cents. Create `GiftCredit(gift=gift, solicitor=solicitor, amount_cents=credit_amount)`.
     - Check for prayer: get `prayer_description` from first row. Call `_maybe_create_prayer_intention(gift, prayer_text, contact, seen_prayers)`.
     - Increment created_count on success.
     - On savepoint exception, rollback savepoint, log error, continue to next group.
8. Create ImportBatch with type=RE_GIFT, status=COMPLETED, all counts, summary with errors/warnings/prayer_count.

Log final summary with created/skipped/error counts.
