**Step 3: Add import_re_recurring_gifts orchestrator**

`import_re_recurring_gifts(file_bytes: bytes, filename: str, uploaded_by: User, owner: User) -> ImportBatch`

Follow the same structure as `import_re_gifts()` from Plan 01 but adapted for RecurringGift:

1. SHA256 dedup via `check_duplicate_import(file_bytes, ImportBatchType.RE_RECURRING_GIFT)`. Return with DUPLICATE status if found.
2. Decode via `decode_csv_bytes()`. On failure, create FAILED ImportBatch.
3. Parse CSV, build `col_map` via `_build_header_mapping(fieldnames, RECURRING_GIFT_HEADER_ALIASES)`.
4. Validate required canonical fields. On missing, create FAILED ImportBatch.
5. Group rows by `gift_id` using `_group_rows_by_id(reader, col_map, 'gift_id')` -- reuse from Plan 01.
6. Build lookups: `_build_solicitor_lookup()`, `_build_fund_lookup()` -- reuse from Plan 01.
7. Process each recurring gift group inside `transaction.atomic()` with savepoints:
   - For each `(gift_id, rows)`:
     - Use savepoint per group.
     - First row provides recurring-gift-level data.
     - Look up Contact by `external_constituent_id`. If not found, skip entire group (same as gifts per CONTEXT.md).
     - Check `RecurringGift.objects.filter(external_gift_id=gift_id).exists()`. If exists, skip group (dedup against DB).
     - Parse amount via `_parse_amount_to_cents()` from first row's `amount`. If 0, error and skip.
     - Parse start_date via `_parse_date()` from first row's `start_date`. If None, error and skip.
     - Parse end_date via `_parse_date()` from first row's `end_date`. None is acceptable (ongoing commitment).
     - Map frequency: get first row's `frequency` field, lowercase, look up in `FREQUENCY_MAP`. If not found and field is non-empty, log error and skip group. If empty, default to `RecurringGiftFrequency.MONTHLY` with a warning.
     - Map status: get first row's `status` field, lowercase, look up in `STATUS_MAP`. If not found and field is non-empty, log error and skip group. If empty, default to `RecurringGiftStatus.ACTIVE`.
     - Match fund: same pattern as gifts (lookup by external_id then name, None if not found).
     - Create `RecurringGift` with `external_gift_id=gift_id`, `donor_contact=contact`, `amount_cents=amount_cents`, `frequency=mapped_frequency`, `start_date=start_date`, `end_date=end_date`, `status=mapped_status`, `fund=matched_fund`, `description=first_row.get('description', '')`.
     - For each row: extract solicitor_name. Normalize via `normalize_solicitor_name()`. Look up in solicitor_lookup. If not found, log error (skip credit). If found, parse credit_amount. If 0, default to gift amount_cents. Create `RecurringGiftCredit(recurring_gift=rg, solicitor=solicitor, amount_cents=credit_amount)`.
     - Increment created_count.
     - On savepoint exception, rollback, log error, continue.
8. Create ImportBatch with type=RE_RECURRING_GIFT, all counts and summary.

NOTE: RecurringGift import does NOT create prayer intentions (prayer text is only on one-time gifts per RE export format and CONTEXT.md scope).
