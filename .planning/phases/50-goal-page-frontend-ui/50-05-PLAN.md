<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 1: Verify Goal page in browser</name>
  <action>All automation complete across Plans 01-04. Human verifies the full Goal page flow in the running application.</action>
  <what-built>
    Complete Goal page at /goal with:
    - Sidebar nav link ("Goal" between Dashboard and Contacts)
    - Goal Settings card: monthly goal input (dollars), goal_weeks input, journal checkbox list, Save Settings button
    - Progress card: three progress bars (Monthly Support, Calls Completed, Meetings Held) with tick marks at 25/50/75/100%; calls and meetings inline inputs; Save Progress button
    - Monthly Support bar color: red below 75%, green 75-99%, amber/gold at 100%+
    - Motivational milestone messages below Monthly Support bar (0/25/50/75/100% thresholds)
    - Empty states: prompt when no goal set; prompt when no journals selected
    - Pacing Targets card: 4 stat tiles (Calls Needed, Meetings Needed, Calls/Week, Meetings/Week)
    - Supervisor/admin read-only mode: inputs disabled, Save buttons hidden, banner shown
  </what-built>
  <how-to-verify>
    Start the dev server: `cd frontend && npm run dev`
    Also ensure backend is running: `python manage.py runserver`

    **Check 1 — Nav link (GOAL-01):**
    Log in as any user. Confirm "Goal" appears in the sidebar between Dashboard and Contacts.

    **Check 2 — Empty states (GOAL-09):**
    Navigate to /goal. If no goal is set: confirm the Monthly Support bar shows a prompt ("Set a goal amount above..."). Save a goal amount — confirm the bar switches to "Select journals above..." if no journals are checked.

    **Check 3 — Goal Settings save:**
    Enter a dollar amount (e.g. 3500), set weeks to 52, check 1-2 journals, click Save Settings. Refresh the page — confirm values are persisted.

    **Check 4 — Progress bars and colors (GOAL-06, GOAL-07):**
    With a goal set and journals selected, observe the Monthly Support bar:
    - If effective support is below 75% of goal: bar should be red
    - If at 75-99%: bar should be green
    - If at 100%+: bar should be amber/gold
    Confirm tick marks are visible at 25%, 50%, 75%, 100% on all three bars.

    **Check 5 — Milestone message (GOAL-08):**
    Observe the text below the Monthly Support bar. It should match the current threshold
    (e.g., "Every journey starts with a first step..." at 0%, "Halfway home!..." at ~50%).

    **Check 6 — Calls/Meetings inputs (GOAL-06):**
    Enter a number for Calls Completed and Meetings Held. Click Save Progress. Refresh — confirm values persist.
    Confirm the Calls and Meetings bars reflect the values relative to pacing targets.

    **Check 7 — Pacing Targets (GOAL-05):**
    With goal = $3,500 and weeks = 52, verify:
    - Calls Needed = ceil((3500/1000) * 25) = 88
    - Meetings Needed = ceil((3500/1000) * 10) = 35
    - Calls/Week = 88/52 = 1.7
    - Meetings/Week = 35/52 = 0.7
    Clear the goal amount — all tiles should show "—".

    **Check 8 — Read-only mode (GOAL-10):**
    Log in as a supervisor or admin user. Navigate to /goal.
    Confirm: all inputs are disabled (cannot type), Save buttons are hidden, and a read-only banner is visible at the top.

    **Check 9 — Responsive layout:**
    Resize browser to mobile width. Confirm pacing tiles stack to single column; layout remains usable.
  </how-to-verify>
  <resume-signal>
    Type "approved" if all checks pass, or describe any issues found (e.g., "Check 4 failed — bar stays red at 100%").
  </resume-signal>
