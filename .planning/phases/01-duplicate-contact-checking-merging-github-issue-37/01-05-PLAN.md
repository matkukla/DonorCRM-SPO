1. Create `frontend/src/pages/contacts/components/MergeFieldRow.tsx`:

```tsx
import { RadioGroup, RadioGroupItem } from "@/components/ui/radio-group"
import { Label } from "@/components/ui/label"
import { TableRow, TableCell } from "@/components/ui/table"

interface MergeFieldRowProps {
  label: string
  fieldName: string
  leftValue: string | null
  rightValue: string | null
  selectedSide: "left" | "right"
  onSideChange: (field: string, side: "left" | "right") => void
  disabled?: boolean
}

export function MergeFieldRow({ label, fieldName, leftValue, rightValue, selectedSide, onSideChange, disabled }: MergeFieldRowProps) {
  const leftDisplay = leftValue || "—"
  const rightDisplay = rightValue || "—"
  const valuesIdentical = (leftValue || "") === (rightValue || "")

  if (valuesIdentical) {
    return (
      <TableRow>
        <TableCell className="font-medium text-sm">{label}</TableCell>
        <TableCell colSpan={2} className="text-center text-sm">{leftDisplay}</TableCell>
      </TableRow>
    )
  }

  return (
    <TableRow>
      <TableCell className="font-medium text-sm">{label}</TableCell>
      <TableCell>
        <RadioGroup value={selectedSide} onValueChange={(v) => onSideChange(fieldName, v as "left" | "right")} disabled={disabled}>
          <div className="flex items-center gap-2">
            <RadioGroupItem value="left" id={`${fieldName}-left`} />
            <Label htmlFor={`${fieldName}-left`} className="text-sm cursor-pointer">{leftDisplay}</Label>
          </div>
        </RadioGroup>
      </TableCell>
      <TableCell>
        <RadioGroup value={selectedSide} onValueChange={(v) => onSideChange(fieldName, v as "left" | "right")} disabled={disabled}>
          <div className="flex items-center gap-2">
            <RadioGroupItem value="right" id={`${fieldName}-right`} />
            <Label htmlFor={`${fieldName}-right`} className="text-sm cursor-pointer">{rightDisplay}</Label>
          </div>
        </RadioGroup>
      </TableCell>
    </TableRow>
  )
}
```

NOTE: The RadioGroup per-field should share the same state. The implementation should use a single RadioGroup wrapping both cells per row OR use a controlled pattern. The simplest correct approach is to NOT use two separate RadioGroups. Instead, use a single RadioGroup per row that spans both cells, with RadioGroupItem in each cell. Alternatively, use plain button/div toggling with the selectedSide state. Choose whichever approach avoids the "two RadioGroups for one value" anti-pattern. A clean approach:

- Use a single `<RadioGroup>` wrapping the entire row content (wrap both cells)
- OR use custom buttons styled as radio-like selectors, managing state via `selectedSide`/`onSideChange`

The preferred approach for table rows is to NOT use RadioGroup inside table cells (breaks table semantics). Instead, use clickable cells with visual radio indicators:
- Each cell is a button with `aria-pressed` or custom radio styling
- The `selectedSide` state controls which cell shows the "selected" indicator (filled circle)
- Use `role="radio"` and `aria-checked` for accessibility

Implement whichever approach compiles cleanly with the Table component.

2. Create `frontend/src/pages/contacts/DuplicateMergeView.tsx` (replacing the placeholder from Plan 04):

**URL parsing:** Extract pairId from useParams, split on `--` to get leftId and rightId.

**Data loading:** Use `useContact(leftId)` and `useContact(rightId)` to load both contact details. Additionally, load related record counts. Since there's no dedicated endpoint for FK counts, use the existing contact sub-endpoints to get counts:
- `useContactDonations(rightId)` -> array.length for gifts count on "right" contact
- `useContactPledges(rightId)` -> array.length for recurring gifts count
- `useContactTasks(rightId)` -> array.length for tasks count
- `useContactJournals(rightId)` -> array.length for journal contacts count

The loser's related records are the ones being migrated, so load counts dynamically based on which side is the loser (the non-survivor).

**State:**
- `survivorSide: "left" | "right" | null` (initially null, set by survivor RadioGroup)
- `fieldOverrides: Record<string, "left" | "right">` (initially all fields set to survivor's side)
- When survivorSide changes, reset all fieldOverrides to the new survivor's side

**Merge execution:**
- Build field_overrides dict: for each field where the override differs from survivor's side, include it
  - If survivorSide is "left", only send field_overrides for fields where value is "right"
  - If survivorSide is "right", only send field_overrides for fields where value is "left"
- survivor_id = survivorSide === "left" ? leftId : rightId
- loser_id = survivorSide === "left" ? rightId : leftId
- Call mergeMutation.mutateAsync({ survivor_id, loser_id, field_overrides })
- On success: toast("Contacts merged successfully.") + navigate("/contacts/duplicates")
- On error: toast.error("Failed to merge contacts. No changes were made.")

**Keep Both:**
- Call dismissMutation with {contact_a_id: leftId, contact_b_id: rightId}
- navigate("/contacts/duplicates")

**Loading state:** Show skeleton cards (h-96 bg-muted rounded animate-pulse) while contacts load, matching ContactForm loading pattern.

**Error state:** If either contact fails to load, show error message with retry.
