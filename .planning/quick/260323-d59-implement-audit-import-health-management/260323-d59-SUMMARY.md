## Implementation Details

### Command: `python manage.py audit_import_health`

**5 Diagnostic Sections:**

1. **Solicitor Linking** -- Counts linked/unlinked solicitors with percentages. Detects near-misses using `difflib.SequenceMatcher` (ratio >= 0.8) comparing unlinked solicitor names against User full names and MissionaryAlias source_names.

2. **Contact Ownership** -- Groups contacts by owner role. Detects misattributions where admin/supervisor-owned contacts have gift credits pointing to missionary-role solicitors.

3. **Gift Credit Integrity** -- Counts total gifts and credits. Identifies orphaned gifts (zero credits) with unlinked dollar value.

4. **Recurring Gift Credit Integrity** -- Same pattern as Section 3 but scoped to active recurring gifts only.

5. **Dashboard Impact Estimate** -- Per-missionary breakdown of contacts, gifts, recurring gifts, and credits. Flags missionaries with 0 contacts or 0 gifts but existing gift credits (data mismatch).

**Flags:**
- `--verbose`: Shows per-record detail lists (unlinked solicitor names, misattributed contacts, orphaned gift details, flagged missionaries with reasons)
- `--json`: Outputs machine-readable JSON with all 5 sections as keys, skipping styled text output

**Verdict:** HEALTHY (0 issues) or NEEDS ATTENTION (N issues) based on sum of: unlinked solicitors + misattributions + orphaned gifts + orphaned recurring gifts + flagged missionaries.

**Edge case:** Zero solicitors prints "No solicitors found. Import solicitors first." and exits cleanly.
