# Requirements: DonorCRM v1.3

**Defined:** 2026-02-16
**Core Value:** Missionaries can manage donor relationships efficiently, with accurate data imported from their organization's systems, and leadership can proactively support their teams through cross-missionary analytics.

## v1.3 Requirements

Requirements for v1.3 — Smartsheet Import, Filters & Polish. All 35 requirements shipped.

### Smartsheet MPD Report Import

- [x] **IMP-01**: Admin can upload an Excel (.xlsx) or CSV Smartsheet MPD report file
- [x] **IMP-02**: System detects file format automatically from content (magic bytes)
- [x] **IMP-03**: System matches each row to an existing DonorCRM user by First Name + Last Name
- [x] **IMP-04**: Unmatched rows are reported to admin (not silently dropped)
- [x] **IMP-05**: System sanitizes formula injection characters (=, +, -, @) in cell values before storage
- [x] **IMP-06**: Admin has a dedicated upload page for the monthly Smartsheet report
- [x] **IMP-07**: Admin sees import results: matched/unmatched missionaries, snapshot counts
- [x] **IMP-08**: Admin dashboard shows latest MPD data per missionary (MPD Cap, Rollover Balance, financial stats)
- [x] **IMP-09**: Each missionary sees their own MPD data in their personal view
- [x] **IMP-10**: Monthly snapshots accumulate historically for trend analysis

### List Page Filters

- [x] **FLT-01**: User can filter by date range (start/end) on donation, journal, pledge, and transaction pages
- [x] **FLT-02**: User can filter by amount range (min/max) on donation and pledge pages
- [x] **FLT-03**: User can filter contacts by group membership
- [x] **FLT-04**: Admin can filter contacts and donations by owner (missionary)
- [x] **FLT-05**: User can filter donations by payment method and fund
- [x] **FLT-06**: User can filter pledges by frequency and search by donor name
- [x] **FLT-07**: User can search and filter journals by name, date range, and archived status
- [x] **FLT-08**: Filter state persists in URL params on all pages (fix Transactions page bug)
- [x] **FLT-09**: User can clear all active filters with one click
- [x] **FLT-10**: User can apply filter presets ("Needs Thank You", "This Month", "Late Pledges", "Stalled")
- [x] **FLT-11**: User sees active filter summary badges showing applied filters
- [x] **FLT-12**: User can export filtered results to CSV
- [x] **FLT-13**: All filtering is server-side with proper query optimization

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| Import undo/rollback | Snapshots are immutable records; admin can re-upload corrected files |
| Automated Smartsheet API | Manual monthly upload is sufficient; API adds auth complexity |
| Real-time import progress | Imports <1s for typical missionary data (10-50 rows) |
| Advanced filter query builder | Overwhelming for 90% of users; stacked AND covers 95% of use cases |
| Saved filters (backend) | URL params already bookmarkable and shareable; no new model needed |
| Multi-select filter dropdowns | Complicates serialization and query logic; marginal value |
| Automated visual regression testing | Overkill for 13 files; manual dark mode checklist sufficient |
| Full OWASP penetration test | Targeted audit of known issues more cost-effective |
| APM monitoring infrastructure | Django Debug Toolbar in dev sufficient; known N+1 patterns |
