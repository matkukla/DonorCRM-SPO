**CURRENCY_FIELDS** -- set of field names that need currency parsing:
All DecimalField field names from MPDSnapshot (14 fields).

**BOOLEAN_FIELDS** -- set of field names that need Yes/No parsing:
`{'met_mpd_standard', 'met_maximum', 'match_met', 'match_met_rest_fy'}`

**PERCENTAGE_FIELDS** -- set: `{'pct_standard_to_max'}`

**MONTHS_REMAINING_FIELD** -- string: `'months_remaining_rf'`

**Functions to implement:**

1. `detect_file_format(file_bytes: bytes) -> str` -- Check first 4 bytes for `PK\x03\x04` (XLSX/ZIP). Return `'xlsx'` or `'csv'`.

2. `sanitize_cell_value(value)` -- Strip formula injection characters from the START of string values. Strip `=`, `+`, `@`, `\t`, `\r` from start. Do NOT strip `-` (negative currency values like `-$468.33` are legitimate). Return the cleaned value. If value is not a string or is None, return as-is.

3. `parse_currency(value) -> Decimal | None` -- Parse currency strings like `"$3,085.00"`, `"-$468.33"`, `"($468.33)"` to Decimal. Handle: `None` returns `None`, `int`/`float` returns `Decimal(str(value))`, empty string returns `None`, `str` strips `$`, `,`, spaces, handles parenthetical negatives `(X)` -> `-X`. Returns `None` on parse failure (not an error for individual cells).

4. `parse_yes_no(value) -> bool | None` -- Parse `"Yes"/"No"` to `True`/`False`. Also accept `"true"/"false"`, `"1"/"0"`. Return `None` for `None` or empty. Case-insensitive.

5. `parse_percentage(value) -> int | None` -- Parse percentage. Handle string `"104%"` -> `104`, `"-16%"` -> `-16`. Handle float from XLSX: if `abs(value) <= 10`, multiply by 100 (e.g., `1.04` -> `104`). Handle int passthrough. Return `None` for None/empty.

6. `parse_months_remaining(value) -> str` -- Return `'infinite'` if string value is `'infinite'` (case-insensitive). If numeric (int/float), format as string with up to 6 decimal places (strip trailing zeros). If string that's numeric, same treatment. Return empty string for None/empty.

7. `build_column_index(headers: list[str]) -> tuple[dict, list[str]]` -- Map header positions to field names using `SMARTSHEET_COLUMN_MAP`. Normalize each header with `.strip().lower()`. Skip columns in `SKIP_COLUMNS`. Return `(index_map, unrecognized_columns)` where `index_map` is `{col_position: field_name}` and `unrecognized_columns` lists headers not found in the map.

8. `parse_xlsx(file_bytes: bytes) -> tuple[list[str], list[list]]` -- Use openpyxl `load_workbook(BytesIO(file_bytes), read_only=True, data_only=True)`, get active sheet, extract headers from first row, data from subsequent rows. Close workbook. Return `(headers, data_rows)`.

9. `parse_csv(file_bytes: bytes) -> tuple[list[str], list[list]]` -- Decode with `utf-8-sig` encoding (handles BOM). Use `csv.reader`. Extract headers from first row, data from remaining rows. Return `(headers, data_rows)`.

10. `parse_row(row_values: list, column_index: dict) -> dict` -- For each position in column_index, extract the value from row_values (handle index out of range as None). Apply sanitize_cell_value to string values first. Then based on field name:
    - If in CURRENCY_FIELDS: `parse_currency(value)`
    - If in BOOLEAN_FIELDS: `parse_yes_no(value)`
    - If in PERCENTAGE_FIELDS: `parse_percentage(value)`
    - If == MONTHS_REMAINING_FIELD: `parse_months_remaining(value)`
    - If starts with `_` (matching keys): just sanitize and strip as string
    Return dict of `{field_name: parsed_value}`.

11. `match_users(parsed_rows: list[dict]) -> tuple[list[tuple], list[dict]]` -- Pre-fetch all active users into a dict keyed by `(first_name.lower().strip(), last_name.lower().strip())`. Detect duplicate name keys and warn (log warning, treat as ambiguous). For each row, look up `(_first_name.lower().strip(), _last_name.lower().strip())`. Return `(matched, unmatched)` where matched is `[(user, row_data_without_underscored_keys), ...]` and unmatched is `[{row: N, first_name: X, last_name: Y}, ...]`. Row numbers start at 2 (header is row 1).

12. `process_mpd_upload(file_bytes: bytes, filename: str, uploaded_by) -> MPDUpload` -- The main orchestrator function:
    a. Detect file format
    b. Create MPDUpload record with status='processing'
    c. Parse file (xlsx or csv)
    d. Build column index from headers; if no recognized columns found, mark upload as failed with error, return
    e. Parse all data rows
    f. Match users
    g. Bulk create MPDSnapshot records for matched rows (use `bulk_create` since there's a unique constraint -- if IntegrityError on individual, fall back to one-by-one)
    h. Update MPDUpload with counts, unmatched_rows, status='completed'
    i. Return the MPDUpload instance

    Wrap the whole thing in a try/except. On exception, mark upload as failed with the error message and re-raise or return the failed upload.

**Important implementation notes:**
- When building snapshot kwargs from parsed row, exclude keys starting with `_` (matching keys like `_first_name`, `_last_name`).
- Use `MPDSnapshot.objects.bulk_create(snapshots, ignore_conflicts=False)` for efficiency. If this raises IntegrityError (e.g., same user appears twice in one file), catch it and fall back to individual creates with try/except per row, logging duplicates.
- Log at INFO level: upload started, format detected, columns mapped, rows parsed, users matched, snapshots created.
- Log at WARNING level: unrecognized columns, duplicate user names in DB, duplicate user rows in file.

   Add the view class after the existing `FundListView` (at the end of the file):

   ```python
   class MPDImportView(APIView):
       """
       POST: Upload Smartsheet MPD Dashboard Report (admin only).

       Accepts CSV or XLSX file. Auto-detects format from file content.
       Matches each row to a DonorCRM user by First Name + Last Name.
       Creates MPDSnapshot records for matched users.

       Response includes:
       - upload_id: UUID of the MPDUpload record
       - status: 'completed' or 'failed'
       - total_rows: number of data rows in file
       - matched_count: number of rows matched to users
       - unmatched_count: number of rows not matched
       - unmatched_rows: list of {row, first_name, last_name} for unmatched
       - snapshot_count: number of MPDSnapshot records created
       """
       permission_classes = [permissions.IsAuthenticated, IsAdmin]
       parser_classes = [MultiPartParser]

       def post(self, request):
           if 'file' not in request.FILES:
               return Response(
                   {'detail': 'No file provided.'},
                   status=status.HTTP_400_BAD_REQUEST
               )

           file = request.FILES['file']
           if file.size > MAX_UPLOAD_SIZE:
               return Response(
                   {'detail': 'File too large (max 10 MB).'},
                   status=status.HTTP_400_BAD_REQUEST
               )

           # Read raw bytes for format detection (not decoding to string)
           file_bytes = file.read()

           try:
               upload = process_mpd_upload(
                   file_bytes=file_bytes,
                   filename=file.name,
                   uploaded_by=request.user,
               )
           except Exception as e:
               logger.error(f'MPD import failed: {e}')
               return Response(
                   {'detail': f'Import failed: {str(e)}'},
                   status=status.HTTP_400_BAD_REQUEST
               )

           response_data = {
               'upload_id': str(upload.id),
               'status': upload.status,
               'total_rows': upload.total_rows,
               'matched_count': upload.matched_count,
               'unmatched_count': upload.unmatched_count,
               'unmatched_rows': upload.unmatched_rows,
               'snapshot_count': upload.snapshots.count(),
           }

           if upload.status == 'failed':
               response_data['error'] = upload.error_message
               return Response(response_data, status=status.HTTP_400_BAD_REQUEST)

           return Response(response_data, status=status.HTTP_201_CREATED)
   ```

   Note: The view does NOT check file extension (`.csv` or `.xlsx`). Format is auto-detected from magic bytes per CONTEXT.md. This is different from existing import views which require `.csv` extension.
