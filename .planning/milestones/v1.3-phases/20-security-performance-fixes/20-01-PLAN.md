  **QAL-07 fix in `apps/pledges/models.py`:**

  In the `monthly_equivalent` property (around line 137-146), replace float arithmetic with Decimal:
  ```python
  @property
  def monthly_equivalent(self):
      """Calculate monthly equivalent for support tracking."""
      from decimal import Decimal
      multipliers = {
          PledgeFrequency.MONTHLY: Decimal('1'),
          PledgeFrequency.QUARTERLY: Decimal('1') / Decimal('3'),
          PledgeFrequency.SEMI_ANNUAL: Decimal('1') / Decimal('6'),
          PledgeFrequency.ANNUAL: Decimal('1') / Decimal('12'),
      }
      return round(self.amount * multipliers.get(self.frequency, Decimal('1')), 2)
  ```

  Note: `self.amount` is already a DecimalField, so `self.amount * Decimal(...)` is Decimal arithmetic. The `round(..., 2)` returns a Decimal rounded to 2 decimal places.

  **No data migration needed** -- `monthly_equivalent` is a `@property` computed on read, not a stored field. The user decision requested "run a data migration to recalculate all existing pledge values" but since there are no stored monthly_equivalent values to recalculate (it's computed on every access), fixing the property is sufficient.

  **Callers that wrap in float()** (dashboard/services.py:147 `float(sum(...))`) will continue to work -- `float(Decimal('100.00'))` returns `100.0` which is fine for JSON serialization at the API boundary.
