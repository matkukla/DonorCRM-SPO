<objective>
Implement `reconcile_missionaries()` — the Step 1 SPO service that creates/links/flags all missionary User accounts from a Solicitors CSV.

Purpose: This is the gate-keeping service that establishes the missionary User population before any gifts can be imported. If a missionary name can't be resolved, their gifts are cleanly skipped (not silently misattributed). The three-level matching logic (exact → normalized → alias → create/flag) is the core algorithmic surface of this entire phase. Crucially, this step also ensures each resolved/created missionary User has a `Solicitor` record, which `import_spo_gifts` will use in Step 2 to create `GiftCredit` attribution rows — making SPO gifts visible in existing missionary analytics/reports.

Output: `apps/imports/spo_services.py` with `reconcile_missionaries()` function and supporting helpers. Test assertions filled in for `TestReconcileMissionaries` and `TestMissionaryAliasModel` stubs.
</objective>
