# Plan 43-05: My Team Page + Missionary Profile Page

## Objective

Build two new pages: `TeamPage.tsx` (team list for supervisor/coach) and `MissionaryProfilePage.tsx` (read-only missionary profile with tabbed content).

## Tasks

### Task 1: Create frontend/src/pages/team/TeamPage.tsx

**Data source:** `user.supervised_users` from auth context. The backend populates this for both supervisor AND coach roles (coach role returns `coached_users` under the same field name — handled in 43-01).

**Layout:**
```
<PageHeader
  title="My Team"
  subtitle={user.role === "coach" ? "Your coachees" : "Missionaries under your supervision"}
/>

<Input placeholder="Search missionaries..." value={search} onChange={...} />

<Table>
  columns: [Name, Email, Action]
  rows: filtered supervised_users
  Action: <Button variant="outline" size="sm" asChild><Link to={`/team/${missionary.id}`}>View Profile →</Link></Button>
</Table>

// Empty state when no missionaries assigned:
<EmptyState message="No missionaries assigned yet" />
```

**Search filter:** filter `supervised_users` by name/email client-side.

Look at existing list pages for PageHeader, Table patterns, empty state patterns.

### Task 2: Create frontend/src/pages/team/MissionaryProfilePage.tsx

**Route:** `/team/:userId`

**Authorization:** Before rendering, verify the userId is in `user.supervised_users`. If not found, render a 403/unauthorized state.

**Data fetching:**
- Fetch contacts, journals, tasks, and (supervisor only) donations filtered to `owner=userId`
- Use existing hooks (`useContacts`, `useJournals`, `useTasks`, `useDonations`) passing owner as filter param
- Read those hook files to understand how filter params work

**Layout:**
```tsx
// Back link
<Link to="/team">← Back to My Team</Link>

// Header
<h1>{missionary.full_name}</h1>
<Badge>{roleLabel}</Badge>
<span>{missionary.email}</span>

// Tabs
<Tabs defaultValue="contacts">
  <TabsList>
    <TabsTrigger value="contacts">Contacts</TabsTrigger>
    <TabsTrigger value="journals">Journals</TabsTrigger>
    <TabsTrigger value="tasks">Tasks</TabsTrigger>
    {!isCoach && <TabsTrigger value="donations">Donations</TabsTrigger>}
  </TabsList>

  <TabsContent value="contacts">
    // Read-only contact list filtered to owner=userId
    // No "Add Contact" button
    // Clicking contact → /contacts/:id (existing isReadOnly guard handles read-only display)
  </TabsContent>

  <TabsContent value="journals">
    // Read-only journal list filtered to owner=userId
    // No "Add Journal" button
  </TabsContent>

  <TabsContent value="tasks">
    // Read-only task list filtered to owner=userId
    // No "Add Task" button
  </TabsContent>

  {!isCoach && (
    <TabsContent value="donations">
      // Read-only donations list filtered to owner=userId
    </TabsContent>
  )}
</Tabs>
```

**Implementation notes:**
- `isCoach = user?.role === "coach"`
- The missionary data (name, email) comes from `user.supervised_users.find(u => u.id === userId)`
- For the tabbed content, reuse existing list components or render simple tables — look at what hook interfaces exist
- Keep it simple: don't recreate full feature pages, just show the data

**Loading/error states:** Show spinner while data loads, show error if userId not in supervised_users.
