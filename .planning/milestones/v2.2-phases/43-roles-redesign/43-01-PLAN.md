# Plan 43-01: DB Migrations + Backend Model Foundation

## Objective

Create a single atomic migration that renames `staff`→`missionary` and `mission_supervisor`→`supervisor` in the DB, adds a `coach` FK to User, and updates all backend model/permission code to use the new role names.

## Tasks

### Task 1: Create migration 0005_roles_redesign.py

**File:** `apps/users/migrations/0005_roles_redesign.py`

Operations in this exact order (data migration BEFORE AlterField to avoid constraint violations):

1. **RunPython** — data migration to rename existing rows:
   ```python
   def forwards(apps, schema_editor):
       User = apps.get_model('users', 'User')
       User.objects.filter(role='staff').update(role='missionary')
       User.objects.filter(role='mission_supervisor').update(role='supervisor')

   def backwards(apps, schema_editor):
       User = apps.get_model('users', 'User')
       User.objects.filter(role='missionary').update(role='staff')
       User.objects.filter(role='supervisor').update(role='mission_supervisor')
   ```

2. **AlterField** — update `role` choices + default:
   ```python
   field=models.CharField(
       choices=[
           ('missionary', 'Missionary'), ('admin', 'Admin'), ('finance', 'Finance'),
           ('read_only', 'Read Only'), ('supervisor', 'Supervisor'), ('coach', 'Coach'),
       ],
       default='missionary',
       max_length=25,
   )
   ```

3. **AddField** — `coach` FK on User:
   ```python
   models.ForeignKey(
       'self', null=True, blank=True, on_delete=models.SET_NULL,
       related_name='coached_users',
       help_text='Coach assigned to this user'
   )
   ```
   Column name: `coach_id`
   Note: `supervisor` FK already exists from Phase 42 migration 0004 (`related_name='supervised_users'`, column `supervisor_id`).

Run `python manage.py migrate` to verify it applies cleanly.

### Task 3: Update apps/core/permissions.py

- Update `get_visible_user_ids()`:
  - Change `'mission_supervisor'` → `'supervisor'` in existing branch
  - Add coach branch after supervisor branch:
    ```python
    if user.role == 'coach':
        ids = set(
            user.coached_users
            .filter(is_active=True)
            .values_list('id', flat=True)
        )
        ids.add(user.id)
        return ids
    ```
  - Change `'staff'` → `'missionary'` in the fallback (it currently returns `{user.id}` for staff, this still applies to missionary)

- Update `IsStaffOrAbove.has_permission()`:
  - Change role list: `['admin', 'finance', 'missionary', 'supervisor']` (do NOT add `'coach'` — coach cannot write financial data, IsStaffOrAbove gates financial writes)

- Add `is_financial_role()` helper:
  ```python
  def is_financial_role(user):
      """Returns True if user can see financial data. Coach is excluded."""
      return user.role in ['admin', 'finance', 'read_only', 'supervisor', 'missionary']
  ```

- Update docstring at top of file to reflect new roles
- Replace all `'mission_supervisor'` → `'supervisor'` and `'staff'` → `'missionary'` string literals

### Task 4: Update apps/users/serializers.py

- In `CurrentUserSerializer.get_supervised_users()`: extend to handle coach:
  ```python
  def get_supervised_users(self, obj):
      if obj.role == 'supervisor':
          qs = obj.supervised_users.filter(is_active=True)
      elif obj.role == 'coach':
          qs = obj.coached_users.filter(is_active=True)
      else:
          return []
      return SupervisedUserSerializer(qs, many=True).data
  ```

  (Keep field name `supervised_users` — frontend already consumes it for both roles)

- Add `coach_id` to `UserSerializer` fields (the FK ID)
- Update all `'staff'`/`'mission_supervisor'` string literals
