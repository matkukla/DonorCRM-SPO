    Create a new admin-only endpoint GET /api/v1/insights/admin/user-drilldown/?user_id={id} that returns a combined summary for quick user inspection.

    **Service function in services.py -- add `get_user_drilldown(user_id)`:**
    - Fetch the User object by user_id (return error dict if not found)
    - Reuse existing query patterns from get_user_performance for per-user stats:
      - total_contacts: Contact.objects.filter(owner_id=user_id).count()
      - active_journals: Journal.objects.filter(owner_id=user_id, is_archived=False).count()
      - decisions_logged: Decision.objects.filter(journal_contact__journal__owner_id=user_id).count()
      - conversion_rate: contacts with decisions / total_contacts * 100
      - total_donations: Donation.objects.filter(contact__owner_id=user_id).aggregate(Sum('amount'))
      - donation_count: Donation.objects.filter(contact__owner_id=user_id).count()
    - Stalled contact count for this user:
      - Use same stalled logic as get_stalled_contacts but filter Contact.objects.filter(owner_id=user_id)
      - Contacts in journals with last_activity_date > 14 days ago or null
    - Recent journals (top 5, non-archived):
      - Journal.objects.filter(owner_id=user_id, is_archived=False).annotate(member_count, decision_count, active_member_count)[:5]
      - Same annotation pattern as get_user_journals
    - Return dict with: user (id, name, email, role), stats (total_contacts, active_journals, decisions_logged, conversion_rate, total_donations, donation_count, stalled_contacts), journals list
    - Target: <15 queries

    **Serializer in serializers.py:**
    - Add UserDrilldownSerializer with nested user info, stats, and journals list

    **View in views.py -- add UserDrilldownView(APIView):**
    - permission_classes: [IsAuthenticated, IsAdmin]
    - Parse user_id query param (required, return 400 if missing)
    - Call get_user_drilldown(user_id=user_id)
    - Return 404 if user not found (service returns error indicator)

    **URL in urls.py:**
    - path('admin/user-drilldown/', UserDrilldownView.as_view(), name='admin-user-drilldown')

    In services.py `get_team_activity`, add `'user_id': str(e.user.id)` to each activity dict. This is needed so the frontend can pass user_id to the drilldown endpoint.

    - Test returns 404 for non-existent user
