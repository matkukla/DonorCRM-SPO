# Milestone v2.0: Import Revamp, Prayer Intentions & Dashboard Polish

**Status:** SHIPPED 2026-02-25
**Phases:** 27-36
**Total Plans:** 27
**Requirements:** 46/46 complete
**Timeline:** 2026-01-13 to 2026-02-25
**Codebase Impact:** 272 files changed, 28,603 insertions(+), 10,040 deletions(-)

## Overview

Replace the existing Donation/Pledge system with Gift/RecurringGift models, build a Raiser's Edge CSV import pipeline, add prayer intentions tracking, and make dashboard tiles draggable. This was the largest milestone to date, touching nearly every layer of the stack — new data models, complete data migration, 4-type RE import pipeline, generic CSV import, prayer intentions feature, dashboard drag-and-drop, and a comprehensive full-stack audit.

## Key Accomplishments

1. **Gift/RecurringGift data model migration** — Replaced Donation/Pledge models entirely with RE-compatible Gift/RecurringGift, including solicitor credit splitting, cents-based amounts, and UUID-preserved data migration
2. **Raiser's Edge CSV import pipeline** — 4-type import (Constituent, Solicitor, Gift, Recurring Gift) with SHA256 dedup, cascading encoding detection, multi-row grouping, and row-level error reporting
3. **Generic CSV import** — Configurable contact/donation import with matching strategies (name, email, external ID) and shared error infrastructure
4. **Prayer Intentions** — Full CRUD with Today's Focus rotation, Focus Mode overlay with keyboard navigation, warm amber chapel aesthetic, and auto-creation from RE gift descriptions
5. **Dashboard polish** — Drag-and-drop tile reordering with dnd-kit, dashboard queries updated to Gift/RecurringGift model
6. **Full-stack audit** — 52 issues addressed across security, performance, code quality, UI/UX, and API consistency, plus 49 new tests

## Milestone Summary

**Key Decisions:**
- REPLACE Donation/Pledge with Gift/RecurringGift (full data migration, not dual-model additive)
- Money fields use PositiveBigIntegerField (cents) with Decimal amount_dollars property
- Solicitor.user changed from OneToOneField to ForeignKey (many-to-one)
- PrayerIntention.gift FK migrated to M2M for multi-gift prayer dedup
- Old SPO import functions removed (410 Gone) since RE pipeline supersedes them
- Desktop-only drag (PointerSensor, no TouchSensor) per user decision
- Generic import uses configurable match_by parameter (name, email, external_id)

**Issues Resolved:**
- ImportBatch duplicate status persistence (Phase 36)
- Gift amount dollar-to-cents filter conversion (Phase 36)
- Dashboard recent gifts field name mismatch (hotfix b2dc4f6)
- NeedsAttention conditional rendering fix (Phase 36)
- React Query cache key corrections (Phase 36)
- 12 bare int() casts replaced with safe param parsing (Phase 36)

**Issues Deferred:**
- Phase 37: Security check (planned, not yet executed)
- Smartsheet openpyxl import error (could not reproduce)
- Dashboard tile persistence to backend (session-only for now)
- Prayer intention reminders/notifications (no notification infrastructure)

**Technical Debt Incurred:**
- Late pledge detection returns empty data gracefully (RecurringGift has no is_late field)
- EventType names preserved as historical labels (DONATION_RECEIVED, PLEDGE_CREATED)
